## 1. 設計主題與視覺風格

### 1.1 主題定義

奶酪大盜是一款社交推理遊戲，v2.0 的設計核心是**「桌上的語音主持設備」**。螢幕設計需要在設備放在桌上時也能清晰辨識階段和倒計時。白天階段偏向溫暖活潑，夜晚階段偏向神秘靜謐，透過明確的日夜視覺切換強化遊戲沉浸感。

### 1.2 設計原則（v2.0 新增）

1. **遠距可讀**：主持進行頁的關鍵資訊（階段名稱、倒計時）在 1-2 公尺距離內可讀
2. **大按鈕優先**：暫停/恢復等操作按鈕要足夠大，方便快速操作
3. **最少干擾**：遊戲進行中螢幕資訊精簡，不分散玩家注意力
4. **語音為主**：UI 是語音的輔助，不是核心

### 1.3 視覺風格關鍵詞

| 維度 | 關鍵詞 | 說明 |
|------|--------|------|
| 整體風格 | 扁平插畫卡通風 | 圓潤線條、柔和陰影、無硬邊 |
| 角色設計 | Q 版老鼠 | 大頭小身、誇張表情、辨識度高 |
| 道具設計 | 奶酪、骰盅、放大鏡 | 寫實細節配卡通輪廓 |
| 場景氛圍 | 歐式小鎮廚房 | 木質紋理、磚牆背景、暖色燈光 |
| 夜晚氛圍 | 月光藍調 | 星空、窗簾剪影、微弱燭光 |

### 1.4 角色視覺設定

| 角色 | 外觀特徵 | 色彩基調 | 使用場景 |
|------|----------|----------|----------|
| 大盜鼠 | 黑色眼罩、披風、狡黠微笑 | 深灰色、紫黑色 | 遊戲封面、結算插圖 |
| 貪睡鼠 | 睡帽、半閉眼、枕頭 | 淡紫色、淺藍色 | 遊戲封面、結算插圖 |
| 共犯鼠 | 條紋衫、鬼祟姿態 | 深綠色、灰色 | 規則說明插圖 |

### 1.5 圖標語言

所有圖標採用填充式 (filled) 風格，線條粗度統一為 2px，圓角 2px。優先使用 Material Icons Rounded 變體。遊戲專屬圖標（奶酪、骰盅、老鼠爪印）需自訂繪製，風格與系統圖標保持一致。

### 2.4 日夜切換過渡

日夜切換不是瞬間跳轉，而是一個 800ms 的漸變過渡動畫。背景色、文字色同步進行 CSS transition，配合背景星星的淡入（進入夜晚）或陽光光線的擴散（進入白天）。

```css
.phase-transition {
  transition: background-color 800ms cubic-bezier(0.65, 0, 0.35, 1),
              color 800ms cubic-bezier(0.65, 0, 0.35, 1);
}
```

## 4. 動畫風格總則

### 4.1 動畫時長分級

```css
:root {
  /* 微互動 */
  --anim-micro: 120ms;    /* 按鈕回饋 */
  --anim-fast: 250ms;     /* 狀態切換 */
  /* 功能動畫 */
  --anim-normal: 400ms;   /* 階段切換 */
  /* 儀式動畫 */
  --anim-dramatic: 800ms; /* 日夜切換 */
  --anim-ceremony: 1200ms; /* 結算揭曉 */
}
```

### 4.2 動畫設計原則

1. **有意義**：每個動畫都服務於遊戲敘事（日夜切換 = 階段轉換，倒計時脈衝 = 時間緊迫感）
2. **不阻塞**：純裝飾動畫不阻擋操作
3. **可跳過**：所有超過 1 秒的儀式動畫，點擊螢幕可跳過
4. **減弱模式**：尊重 `prefers-reduced-motion`，動畫降級為淡入淡出

### 5.3 主持進行頁（核心頁面）

**用途**：語音主持的核心介面。整合夜晚叫號、白天討論計時、投票計時、結算所有階段。這是整個 App 最重要的頁面。

**設計理念**：設備放桌上後，螢幕朝向玩家或面朝下。頁面需要在遠距離（1-2 公尺）也能看清當前階段和倒計時。按鈕要大，方便快速暫停。

| 號碼進度條 | 橫向節點連線 | 圓點 + 連線，當前節點脈衝 |
| 5 秒倒數條 | 水平進度條 | 每個叫號後 5 秒倒數 |
| 音量滑桿 | 底部, 全寬 | 即時調整語音音量 |
| 勝負選擇按鈕 | 兩個大按鈕並排 | 結算時組織者選擇 |
| 再來一局按鈕 | primary 樣式 | 保留人數設定重新開始 |

#### 互動說明

- **夜晚階段**：TTS 語音自動依序播報 1-6 號，每號間隔 5 秒
- **白天階段**：自動開始倒數計時，最後 30 秒文字變警告色
- **投票階段**：語音播報「三、二、一，指！」引導面對面投票
- **結算階段**：組織者選擇勝負結果，觸發對應語音和視覺效果
- **暫停/恢復**：任何階段可暫停，暫停時語音和計時同步停止
- **上一步/下一步**：手動跳過或回退某個階段
- **日夜色彩切換**：進入/離開夜晚時 800ms 漸變動畫
- **計時結束**：自動播放語音提示並切換到下一階段

## 6. 關鍵互動設計

### 6.1 夜晚叫號流程（核心互動）

```
叫號循環 (每個號碼):
  1. 號碼數字切換動畫 (250ms, 上一個淡出 + 新數字淡入放大)
  2. TTS 播報「骰子數字是 X 的玩家，請睜開眼睛」
  3. 5 秒倒數（進度條從滿到空）
  4. 過渡到下一個號碼 (250ms)

夜晚完整序列:
  1. 背景色切換至夜間 (800ms)
  2. TTS「請所有人閉上眼睛，夜晚降臨」
  3. 1 號叫號 → 5 秒 → 2 號叫號 → ... → 6 號叫號 → 5 秒
  4. TTS「天亮了，請所有人睜開眼睛」
  5. 背景色切換至日間 (800ms)
  6. 自動進入白天討論階段

TTS 與動畫同步:
  - TTS 開始說話前 200ms 啟動數字切換動畫
  - TTS 結束後啟動 5 秒倒計時
  - 使用 TTS onend 事件控制流程
```

### 6.2 暫停/恢復手勢

```
觸發方式: 點擊暫停按鈕 (右上角 64x64px)

暫停效果:
  - TTS 語音停止（記錄當前進度）
  - 背景音樂停止
  - 倒計時暫停
  - 螢幕顯示半透明遮罩 + 「暫停中」文字
  - 遮罩上顯示大按鈕「繼續遊戲」

恢復效果:
  - 遮罩淡出 (250ms)
  - 從暫停處繼續 TTS 和計時
  - 背景音樂恢復
```

### 6.3 階段切換動畫

```
日 → 夜 切換:
  - 背景色從暖米白漸變至深夜空 (800ms)
  - 星星元素淡入 (CSS animation)
  - 文字色同步過渡

夜 → 日 切換:
  - 背景色從深夜空漸變至暖米白 (800ms)
  - 星星淡出
  - 光線擴散效果 (CSS radial-gradient animation)

倒計時警告:
  - 最後 30 秒：倒計時文字色變為 warning 橘色
  - 最後 10 秒：文字脈衝閃爍 (opacity 0.5-1.0 循環)
```
