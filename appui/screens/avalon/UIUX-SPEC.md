## 1. 頁面清單與流程圖

### 1.1 頁面清單（8 個核心頁面/狀態）

| 編號 | 頁面 | 對應路由 | 說明 |
|------|------|---------|------|
| P1 | 遊戲設置頁 | `/avalon/setup` | 人數、角色配置、語速音量 |
| P2 | 角色分配頁（傳閱） | `/avalon/play` 內部狀態 | 逐人傳閱翻牌查看角色 |
| P3 | 主遊戲頁（任務進度板） | `/avalon/play` | 5 輪任務進度 + 當前階段 |
| P4 | 隊長提名頁 | `/avalon/play` 內部狀態 | 隊長選擇隊員 |
| P5 | 公開投票頁 | `/avalon/play` 內部狀態 | 全員贊成/反對 |
| P6 | 任務執行頁 | `/avalon/play` 內部狀態 | 被選隊員秘密出牌 |
| P7 | 刺殺梅林頁 | `/avalon/play` 內部狀態 | 邪惡方指定梅林 |
| P8 | 遊戲結果頁 | `/avalon/play` 內部狀態 | 勝負揭曉 + 角色公開 |

**設計決策**：與奶酪大盜相同，阿瓦隆採用單一 `AvalonPlayView` 作為核心頁面，內部以狀態機切換子階段。避免路由跳轉導致的頁面閃爍，日夜/階段切換動畫更流暢。

```
    |         |

    |              |
```

## 2. 阿瓦隆專屬色彩系統

### 2.1 色彩方案

阿瓦隆主題是亞瑟王傳說、中世紀奇幻，需要**莊嚴、神秘、史詩感**。以 CSS 變數覆蓋方式實作。

```css
[data-game="avalon"] {
  /* 預設態 */
  --color-primary: #4A6FA5;         /* 亞瑟藍 */
  --color-primary-light: #6B8FBF;
  --color-primary-dark: #2E4A73;
  --color-primary-surface: #1A2332;
  --color-bg: #0F1620;              /* 深夜石牆 */
  --color-bg-surface: #1A2332;
  --color-bg-card: #1E2A3A;
  --color-bg-card-variant: #243042;
  --color-text: #E8E4D9;            /* 羊皮紙白 */
  --color-text-secondary: #A0A0B8;
  --color-text-muted: #6B6B82;
  --color-text-on-primary: #FFFFFF;
  --color-border: #2E3A4E;
  --color-accent: #D4AF37;          /* 聖杯金 */
  --color-accent-light: rgba(212, 175, 55, 0.2);

  /* 陣營色 */
  --color-good: #4A90D9;            /* 正義藍 */
  --color-good-surface: rgba(74, 144, 217, 0.15);
  --color-evil: #C0392B;            /* 邪惡紅 */
  --color-evil-surface: rgba(192, 57, 43, 0.15);

  /* 任務狀態色 */
  --color-mission-success: #27AE60;
  --color-mission-fail: #C0392B;
  --color-mission-pending: #4A6FA5;
  --color-mission-current: #D4AF37;
}

/* 階段色覆蓋 */
[data-game="avalon"][data-phase="nominate"] {
  --color-accent: #D4AF37;          /* 聖杯金 — 權力 */
}
[data-game="avalon"][data-phase="vote"] {
  --color-bg: #1A1520;              /* 偏紫 — 裁決 */
  --color-accent: #9B59B6;
}
[data-game="avalon"][data-phase="mission"] {
  --color-accent: #2ECC71;          /* 行動綠 */
}
[data-game="avalon"][data-phase="assassinate"] {
  --color-bg: #1A0A0A;              /* 血色深紅 */
  --color-accent: #E74C3C;
}
```

### 2.2 對比度驗證

|------|--------|------|
| `--color-text` / `--color-bg` | 13.2:1 | AA |
| `--color-text-secondary` / `--color-bg` | 5.8:1 | AA |
| `--color-accent` / `--color-bg` | 6.1:1 | AA |
| `--color-good` / `--color-bg-card` | 4.8:1 | AA |
| `--color-evil` / `--color-bg-card` | 4.6:1 | AA |

## 3. 核心 UI 元件設計

### 3.1 任務進度追蹤器 (MissionTracker)

整局遊戲始終可見。5 個任務圓盾排成一行。

```
  [ 1 ]---[ 2 ]---[ 3 ]---[ 4 ]---[ 5 ]
   (成功)  (失敗)  (當前)  (待定)  (待定)
    綠盾    紅盾    金框    灰盾    灰盾
    2人     3人     2人     3人     3人
```

| 元素 | 規格 |
|------|------|
| 圓盾直徑 | 48px（待定）/ 56px（當前） |
| 連接線 | 寬 16px, 高 3px |
| 任務編號 | Space Grotesk 20px Bold |
| 隊員人數 | Noto Sans TC 12px |
| 整體高度 | 約 80px |

**狀態視覺**：
- **成功**：綠色填充 (#27AE60)，盾牌內顯示勾號
- **失敗**：紅色填充 (#C0392B)，盾牌內顯示叉號
- **當前**：金色邊框脈衝動畫 (#D4AF37)，顯示任務編號
- **待定**：灰色半透明 (#2E3A4E)，顯示任務編號
- **第 4 輪特殊**：7 人以上局下方標註「需 2 失敗」(warning color)

### 3.2 玩家選擇器 (PlayerCircle)

隊長提名隊員時使用。玩家圍成一圈排列（模擬圓桌）。

```
         [ P1 ]
      [ P8 ]  [ P2 ]
    [ P7 ]      [ P3 ]
      [ P6 ]  [ P4 ]
         [ P5 ]
```

| 元素 | 規格 |
|------|------|
| 玩家圓形頭像 | 56x56px |
| 選中態邊框 | 3px solid #D4AF37 |
| 隊長標記 | 皇冠 SVG 20x20px（右上角） |
| 圓形排列直徑 | 約 260px |
| 選中計數 | 底部 16px |

### 3.3 投票卡片 (VoteCard)

贊成/反對兩張卡片並排，傳閱模式每人選一張。

| 元素 | 規格 |
|------|------|
| 卡片寬度 | 各佔螢幕 45% |
| 卡片高度 | 200px |
| 選中態 | 邊框加粗 + 放大 1.05x + 光暈 |
| 未選態 | opacity 0.6 |

### 3.4 任務執行卡片 (MissionCard)

成功/失敗兩張卡片。正義方「失敗」卡片 disabled。

**隱私關鍵設計**：
- 邪惡方「成功」和「失敗」按鈕**位置隨機化**（左右互換），防止旁人從手指方向推斷
- 出牌前「請確認周圍無人窺視」提示
- 出牌後立即遮罩

### 3.5 角色卡片 (RoleRevealCard) — 翻牌效果

CSS 3D 翻轉，長按 0.5 秒觸發。

**正面（蓋牌態）**：深藍底 + 阿瓦隆徽章 + 「長按查看角色」

**反面（揭示態）**：
- 正義方：藍色底
- 邪惡方：紅色底
- 顯示：陣營、角色插圖、角色名、特殊資訊

**動畫規格**：
- `transform: rotateY(180deg)` + `perspective(1000px)`
- 翻轉時長 400ms
- 自動蓋牌倒計時 5 秒

### 3.6 否決計數器 (RejectCounter)

```
  [ ] [ ] [ ] [ ] [!]
   1   2   3   4   5
```

- 已否決填充紅色
- 第 5 格帶骷髏/警告圖示
- 第 4 次否決時全體脈衝警告色

### 3.7 傳閱遮罩 (PassDeviceOverlay)

所有傳閱環節（角色查看、投票、任務出牌）共用。

- 全屏深色覆蓋 `--color-bg`
- 大字提示「請將設備傳給下一位玩家」
- 確認按鈕才允許進入下一步
- 過渡 250ms fade

## 6. 互動流程設計

### 6.1 傳閱查看角色

```
[語音] "現在進行角色分配，請將設備傳給第一位玩家"
  |
  v
[迴圈: 對每位玩家]
  1. 遮罩：「玩家 X，請確認只有你能看見螢幕」→ [我準備好了]
  2. 蓋牌態角色卡：「長按查看你的角色」
  3. 長按 0.5 秒 → 翻牌 (400ms) → 顯示角色 + 倒計時 5 秒
  4. 倒計時歸零 / 點「蓋牌」→ 翻回正面
  5. 遮罩：「請將設備面朝下傳給下一位」→ [已傳遞]
  |
  v
[語音] "所有人已查看角色，遊戲即將開始"
```

**設計要點**：長按觸發（防誤觸）、自動蓋牌（防遺忘）、傳遞確認（防洩漏）。

### 6.2 秘密任務投票

```
[語音] "任務開始，隊員依序拿起設備"
  |
  v
[迴圈: 對每位隊員]
  1. 遮罩：「隊員 玩家 X，請確認周圍無人窺視」
  2. 出牌選擇（正義方: 只能成功 / 邪惡方: 可選成功或失敗）
  3. 確認 → 遮罩「已出牌，傳給下一位」
  |
  v
洗牌動畫 (800ms) → 逐張翻開 (400ms/張)
[語音] "任務成功/失敗！"
```

### 6.3 刺殺梅林

```
[語音] "正義方三勝，但邪惡陣營有最後機會..."
  → 刺客選擇目標（僅正義方玩家）
  → 二次確認（不可撤銷）
  → 懸念等待 2 秒 + 緊張音效
  → 被選玩家卡片翻轉揭曉角色 (800ms)
  → 命中梅林: 邪惡逆轉勝 / 未命中: 正義方確認勝利
```
