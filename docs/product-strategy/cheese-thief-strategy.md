## 1. 為什麼選擇奶酪大盜作為首發遊戲

### 1.1 策略選擇理由

選擇奶酪大盜作為「桌遊語音主持人」的首發遊戲模板，是基於以下三個維度的綜合評估：

#### A. 遊戲複雜度適中 -- 完美展示語音主持價值

| 評估維度 | 奶酪大盜 | 狼人殺 | 阿瓦隆 |
|----------|---------|--------|--------|
| 規則複雜度 | 低 | 高 | 中高 |
| 主持人負擔 | 中 | 極高 | 高 |
| 單局時間 | 10分鐘 | 30-60分鐘 | 20-40分鐘 |
| 玩家人數 | 4-8人 | 8-18人 | 5-10人 |
| 新手友好度 | 極高 | 中 | 中 |

奶酪大盜的核心機制是「天黑閉眼 + 骰子數字 + 偷奶酪」，規則本身簡單直觀，但**夜晚階段必須有人主持引導每位玩家依序睜眼**。這個特性讓語音主持人的價值一目了然：

- 規則簡單 = 新用戶不會因為不懂遊戲而放棄
- 需要主持人 = 語音主持的功能價值容易被感知
- 單局時間短 = 用戶可以快速體驗完整流程，降低首次使用門檻

#### B. 目標用戶匹配度高

奶酪大盜的用戶畫像與我們的目標用戶高度重疊：

- **小社團聚會**：4-8人正好是朋友聚餐、團建的常見人數
- **輕度桌遊玩家**：不需要複雜的策略知識，派對聚會即可玩
- **桌遊新手**：作為社交推理入門款，門檻極低
- **家庭用戶**：美術風格可愛，適合親子互動

#### C. 語音主持的價值展示清晰

這個流程的痛點非常明確：
1. **需要一人犧牲不玩**：有人當主持就不能參與遊戲
2. **主持節奏影響體驗**：新手主持容易出錯或節奏拖沓
3. **語音氛圍很重要**：「天黑請閉眼」的語氣和節奏直接影響沉浸感

**一台手機放桌上就能自動主持**，完美解決以上三個痛點，讓用戶在第一次使用時就能感受到「啊，這比人工主持方便多了」的價值。

### 1.2 產品定位（v2.0 更新）

```
產品定位 = 「離線 H5 語音主持工具」

核心體驗：
  一台手機/平板放在桌上 → 掃碼開啟網頁 → 選擇遊戲
  → TTS 語音自動主持夜晚、白天、投票全流程
  → 玩家用實體骰子和角色卡
  → App 只負責語音 + 計時 + 音效

差異化定位：
  - 無需安裝：H5 PWA，掃碼即用
  - 離線可用：Service Worker 快取，露營/密室等場景不受網路限制
  - 不需要帳號：打開就用，零門檻
  - 實體道具搭配：尊重桌遊的面對面社交體驗
```

### 1.3 為什麼選擇 H5 PWA 而不是原生 App

| 維度 | 原生 App (Flutter) | H5 PWA (Vue 3) |
|------|-------------------|----------------|
| 分發方式 | App Store / Google Play 上架 | 掃碼即用，無需安裝 |
| 審核週期 | 1-7 天 | 即時部署 |
| 使用門檻 | 搜尋 → 下載 → 安裝 → 開啟 | 掃碼 → 開啟 |
| 開發成本 | 高（iOS + Android 適配） | 低（跨平台） |
| 離線能力 | 原生支援 | Service Worker 支援 |
| TTS 能力 | 系統 TTS | Web Speech API |
| 適合場景 | 重度使用、頻繁開啟 | 輕量使用、掃碼開始 |

**結論**：對於「聚會時拿出手機掃碼開始玩」的使用場景，H5 PWA 的「掃碼即用」體驗遠優於原生 App 的「下載安裝」流程。

## 2. 目標用戶畫像

### 2.1 Persona 1：小社團組織者 (40%)

| 屬性 | 描述 |
|------|------|
| 年齡 | 22-35 歲 |
| 身份 | 上班族、大學生 |
| 場景 | **小社團線下桌遊聚會**、朋友聚餐後、宿舍活動 |
| 痛點 | 想玩需要主持人的桌遊但沒人願意當主持；不想為了一個遊戲下載 App |
| 動機 | 想要一個「掃碼就能用」的語音主持工具 |
| 付費意願 | 中（願意為進階功能付小額費用） |
| 設備 | 手機為主 |

**典型場景**：小明約了 6 個朋友週五在桌遊咖啡廳聚會，帶了奶酪大盜實體遊戲但沒人想當主持人。拿出手機掃桌上的 QR Code，直接開始語音主持，所有人都能參與遊戲。

### 2.2 Persona 2：桌遊咖啡廳老闆 (20%)

| 屬性 | 描述 |
|------|------|
| 年齡 | 25-40 歲 |
| 身份 | 桌遊咖啡廳經營者 |
| 場景 | **店內提供給客人使用的主持工具** |
| 痛點 | 客人不懂規則需要店員解說；忙碌時段無法安排人力當主持 |
| 動機 | 免費的自動主持工具，提升客戶體驗 |
| 付費意願 | 中高（願意為店面專用功能付費） |
| 設備 | 店內平板 |

**典型場景**：桌遊咖啡廳在每張桌上放一個 QR Code，客人自行掃碼就能開始有語音主持的奶酪大盜，不需要店員全程教學和主持。

### 2.3 Persona 3：露營團 / 家庭聚會 (25%)

| 屬性 | 描述 |
|------|------|
| 年齡 | 25-45 歲 |
| 身份 | 露營愛好者、家庭活動組織者 |
| 場景 | **露營地、渡假村、家庭聚會** — 通常無網路環境 |
| 痛點 | 露營地沒網路，App 不能用；帶桌遊但沒人會主持 |
| 動機 | 離線可用的語音主持工具 |
| 付費意願 | 中（願意為好體驗付費） |
| 設備 | 手機 / 平板 |

**典型場景**：週末露營，晚上大家圍坐營火旁想玩桌遊。事先在有網路時訪問過 PWA 並加入主畫面，露營地沒網路也能正常使用語音主持。

### 2.4 Persona 4：桌遊新手 (15%)

| 屬性 | 描述 |
|------|------|
| 年齡 | 18-30 歲 |
| 身份 | 學生、社會新鮮人 |
| 場景 | 第一次玩桌遊、被朋友帶入坑 |
| 痛點 | 不懂規則、怕出糗 |
| 動機 | App 能教規則又能主持，降低社交焦慮 |
| 付費意願 | 低（先免費體驗） |
| 設備 | 手機為主 |

## 3. 競品分析

### 3.1 直接競品

| 競品 | 類型 | 平台 | 優勢 | 劣勢 | 定價 |
|------|------|------|------|------|------|
| 狼人殺AI主持 | 專用AI法官 | iOS/Android | 三語支持、角色豐富 | 需安裝App、僅限狼人殺 | 免費+內購 |
| Werewolf Local Hunt | 面對面狼人殺 | iOS/Android | 面對面遊玩 | 需安裝、僅限狼人殺 | 免費+廣告 |
| 各種線上桌遊平台 | 多遊戲 | Web/App | 遊戲種類多 | 線上對戰非面對面 | 各異 |

### 3.2 間接競品

| 競品 | 類型 | 與我們的關係 |
|------|------|-------------|
| 桌遊實體產品 | 傳統桌遊 | 互補關係，我們是實體遊戲的「主持人配件」 |
| 劇本殺App | 沉浸式推理 | 部分用戶重疊，但定位不同 |
| YouTube 主持影片 | 影片主持 | 替代方案但不互動、不計時 |

### 3.3 我們的差異化優勢（v2.0 更新）

| 差異化維度 | 競品現狀 | 我們的做法 |
|-----------|---------|-----------|
| **分發方式** | 需下載安裝 App | **掃碼即用**，H5 PWA 零安裝 |
| **離線能力** | 多數需要網路 | **完全離線可用**，露營/密室場景 |
| **使用門檻** | 需要帳號、建房間 | **無需帳號**，打開就用 |
| **遊戲覆蓋** | 大多只支持狼人殺 | 平台化設計，支持多種桌遊模板 |
| **與實體道具搭配** | 完全數位化 | **尊重桌遊社交**，搭配實體道具 |
| **本地化** | 多為簡中或英文 | 繁中優先，支持台灣用語 |

### 3.4 核心競爭壁壘

```
短期壁壘 (0-6個月)：
  - 「掃碼即用」的產品體驗差異化
  - 離線可用的獨特場景覆蓋
  - 奶酪大盜首發搶佔心智

中期壁壘 (6-18個月)：
  - 遊戲模板庫的豐富度
  - 桌遊咖啡廳合作網絡
  - 用戶口碑積累

長期壁壘 (18個月+)：
  - 平台化生態（用戶自製模板）
  - 品牌認知：「桌遊語音主持 = 我們」
```

## 4. MVP 範圍定義

### 4.1 P0 -- 必須有（MVP 核心）

| 功能 | 描述 | 驗收標準 | 優先理由 |
|------|------|---------|---------|
| 遊戲選擇 | 選擇奶酪大盜模板 | 點擊即進入設定頁 | 核心流程入口 |
| 人數設定 | 設定 4-8 人玩家數 | 角色組成和物品清單自動更新 | 核心參數 |
| TTS 語音主持 - 夜晚 | 依序叫號 1-6，每號間隔 5 秒 | 語音清晰、節奏穩定、計時精準 | **產品核心價值** |
| TTS 語音主持 - 白天 | 討論倒計時，結束提醒 | 計時準確、語音提示清晰 | 產品核心價值 |
| TTS 語音主持 - 投票 | 投票倒計時，「三二一指」 | 節奏明確 | 產品核心價值 |
| 暫停/恢復 | 暫停語音和計時 | 按鈕夠大、反應即時 | 必要操作 |
| 結算語音 | 播報勝利方 | 組織者選擇結果觸發語音 | 遊戲完整性 |
| 規則教學語音 | TTS 播報規則摘要 | 新手 2 分鐘理解規則 | 降低使用門檻 |
| PWA 離線支援 | Service Worker 快取 | 離線完整遊戲可用 | 核心差異化 |
| 日夜視覺切換 | 夜晚/白天色彩切換 | 800ms 過渡動畫 | 沉浸感 |

### 4.2 P1 -- 應該有（上線後 1-2 個月）

| 功能 | 描述 | 優先理由 |
|------|------|---------|
| 背景音樂/音效 | 夜晚氛圍、白天氛圍、計時音效 | 提升沉浸感 |
| 語速/音量調整 | 多檔位語速和獨立音量控制 | 適應不同場景 |
| 遊戲歷史記錄 | localStorage 記錄每局結果 | 增加黏性 |
| PWA 安裝引導 | 「加入主畫面」提示 | 提升留存 |
| 上一步/下一步 | 手動控制階段跳轉 | 靈活性 |

### 4.3 P2 -- 可以之後做（3 個月後）

| 功能 | 描述 | 優先理由 |
|------|------|---------|

## 5. 商業模式

### 5.1 定價哲學（v2.0 更新）

```
免費基礎版（引流 + 驗證）
    |
    v
付費進階語音/音效包（輕量變現）
    |
    v
付費進階遊戲模板（核心變現）
```

### 5.2 奶酪大盜定價策略：免費

**結論：奶酪大盜基礎版必須完全免費。**

| 理由 | 說明 |
|------|------|
| 降低試用門檻 | H5 PWA + 免費 = 掃碼即用零成本，用戶轉化率最高 |
| 展示核心能力 | 免費完整體驗語音主持的品質 |
| 口碑傳播燃料 | 「免費掃碼就能玩」是最強推廣話術 |
| 桌遊咖啡廳合作 | 店面免費使用，願意推薦給客人 |

### 5.3 變現路徑（v2.0 更新）

#### Phase 1：免費增長期 (0-6個月)

```
收入來源：無（專注用戶增長和口碑建立）

免費內容：
- 奶酪大盜完整語音主持
- 基礎 TTS 語音
- 基本音效
- 離線使用
- 遊戲歷史記錄
```

#### Phase 2：輕量變現期 (6-12個月)

```
收入來源：付費內容包

付費語音包：
- 趣味語音風格 -- NT$60 / RMB 12（搞笑/恐怖/兒童風格）
- 方言語音包 -- NT$60 / RMB 12（台語/粵語主持）

付費音效包：
- 氛圍音效進階版 -- NT$60 / RMB 12（更多背景音樂和音效）

付費遊戲模板：
- 狼人殺語音主持 -- NT$90 / RMB 18
- 阿瓦隆語音主持 -- NT$90 / RMB 18
```

#### Phase 3：訂閱制 (12個月+)

```
免費版：
- 奶酪大盜 + 1款輪替免費遊戲模板
- 基礎語音風格
- 基本音效

訂閱版 -- NT$99/月 或 NT$599/年 (RMB 20/月 或 RMB 120/年)：
- 全遊戲模板庫
- 全語音風格包
- 全音效包
- 遊戲統計面板
- 搶先體驗新遊戲
```

## 6. 成功指標 (KPI)

### 6.1 北極星指標

**週活躍遊戲局數 (Weekly Active Game Sessions)**

> 定義：過去 7 天內完成的遊戲局數（從開始到結算的完整遊戲）

選擇理由：
1. 反映產品核心價值是否被實現
2. 一局遊戲涉及多位玩家，天然代表用戶參與度
3. 「完成遊戲」比「打開頁面」更能反映真實價值

### 6.2 核心指標看板

#### 用戶活躍度

| 指標 | 定義 | MVP 階段目標 (3個月) | 成長階段目標 (6個月) |
|------|------|---------------------|---------------------|
| DAU | 日活躍用戶 | 100 | 1,000 |
| WAU | 週活躍用戶 | 500 | 5,000 |
| MAU | 月活躍用戶 | 1,500 | 15,000 |
| DAU/MAU | 用戶黏性比 | > 10% | > 15% |

#### 遊戲指標

| 指標 | 定義 | 目標 |
|------|------|------|
| 遊戲完成率 | 開始到完成結算的比率 | > 85% |
| 平均每次遊戲局數 | 單次開啟玩幾局 | > 2局 |
| 重玩率 | 結算後點「再來一局」的比例 | > 60% |
| PWA 安裝率 | 訪問用戶中安裝 PWA 的比例 | > 25% |

#### 留存指標

| 指標 | 定義 | 目標 |

#### 品質指標

| 指標 | 定義 | 警戒線 |

| 離線可用率 | 離線完整完成一局 | 100% |
| 首次載入時間 | 首次訪問到可互動 | < 3 秒 |
| 頁面崩潰率 | 遊戲中崩潰比例 | < 0.1% |

### 6.3 關鍵漏斗

## 7. GTM (Go-to-Market) 策略

### 7.1 核心推廣策略：QR Code 裂變

H5 PWA 最大的優勢是「掃碼即用」，這意味著推廣方式與原生 App 完全不同：

```
推廣核心 = 讓 QR Code 出現在目標用戶會看到的地方

物理 QR Code:
  - 桌遊咖啡廳桌面
  - 桌遊實體產品包裝內
  - 露營用品店
  - 大學社團佈告欄

數位 QR Code:
  - 桌遊社群貼文
  - 社群媒體分享
  - 桌遊 YouTuber 影片描述欄
```

### 7.2 整體 GTM 時程

```
Phase 0: 封測期     (上線前1個月)    目標: 30位種子用戶
Phase 1: 軟上線     (Month 1)       目標: 300位用戶
Phase 2: 正式推廣   (Month 2-3)     目標: 3,000位用戶
Phase 3: 規模增長   (Month 4-6)     目標: 15,000位用戶
```

### 7.3 Phase 0：封測期（上線前1個月）

**目標**：收集真實用戶回饋，打磨核心體驗

| 策略 | 具體行動 | 目標人數 |
|------|---------|---------|
| 桌遊社群招募 | PTT BoardGame 板、Dcard 桌遊群組發佈封測 | 15人 |
| 桌遊咖啡廳合作 | 聯繫 2-3 家桌遊咖啡廳提供封測 | 10人 |
| 親友測試 | 產品團隊和朋友圈試玩 | 5人 |

封測重點收集：
- TTS 語音品質滿意度
- 「掃碼即用」體驗是否順暢
- 離線功能是否正常
- 遊戲流程是否完整
- 願不願意推薦給朋友（NPS）

### 7.4 Phase 1：軟上線（Month 1）

**目標**：驗證產品市場契合度 (PMF)

| 管道 | 策略 | 預算 | 預估效果 |
|------|------|------|---------|
| 桌遊咖啡廳 | 在合作店面桌上放 QR Code 立牌 | NT$2,000(物料) | 100 用戶/月 |
| PTT / Dcard | 發佈「掃碼就能玩桌遊」體驗分享 | NT$0 | 80 用戶 |
| Facebook 桌遊社團 | 在社團分享 QR Code | NT$0 | 60 用戶 |
| 桌遊 YouTuber | 1 位小型 YouTuber 試玩 | NT$3,000 | 60 用戶 |

**PMF 判斷標準**：
- 遊戲完成率 > 80%
- D7 留存 > 12%
- NPS > 30
- 桌遊咖啡廳回購意願 > 70%

### 7.5 Phase 2：正式推廣（Month 2-3）

**目標**：放大增長，建立桌遊咖啡廳合作網絡

| 管道 | 策略 | 預算/月 | 預估效果 |
|------|------|--------|---------|
| 桌遊咖啡廳拓展 | 拓展至 10-15 家桌遊咖啡廳 | NT$5,000(物料) | 500 用戶/月 |
| 桌遊 KOL | 3 位桌遊 YouTuber 試玩 | NT$10,000 | 800 用戶/月 |
| IG/FB 廣告 | 投放「掃碼即用」體驗影片 | NT$10,000 | 600 用戶/月 |
| 小紅書 | 桌遊聚會體驗圖文 | NT$3,000 | 300 用戶/月 |
| SEO | 「奶酪大盜怎麼玩」等文章 | NT$0(內部) | 200 用戶/月 |

### 7.6 Phase 3：規模增長（Month 4-6）

| 策略 | 具體行動 |
|------|---------|
| QR Code 裂變 | App 內生成分享 QR Code，方便用戶轉發 |
| 桌遊產品合作 | 與桌遊出版商合作，在實體產品中附 QR Code |
| 露營社群推廣 | 針對露營/戶外社群推廣「離線可用」特性 |
| 第二款遊戲上線 | 狼人殺語音主持模板上線，擴大用戶群 |
| 線下活動 | 參與桌遊展會，現場體驗 |

## 附錄

### A. 風險與緩解

| 風險 | 可能性 | 影響 | 緩解策略 |
|------|--------|------|---------|
| 瀏覽器 TTS 品質不一致 | 高 | 高 | 提供預錄音檔備援；測試主流瀏覽器 |
| PWA 在 iOS Safari 支援有限 | 中 | 中 | 降級方案：純網頁模式仍可用 |
| 用戶不理解「掃碼即用」概念 | 低 | 中 | QR Code 旁附使用說明；製作 30 秒影片 |
| 離線快取失敗 | 低 | 高 | 啟動時檢查快取狀態；提供手動快取按鈕 |
| 競品複製 H5 模式 | 低 | 中 | 快速迭代遊戲庫；建立桌遊咖啡廳合作壁壘 |
