## 執行摘要

### 產品定位
Cheese-Thief 是一款專注於幫助用戶戒除不健康飲食習慣的 Flutter App，採用正向心理學設計，透過每日打卡、渴望管理、成就系統等功能，協助用戶建立健康飲食習慣。

### 核心策略要點

| 面向 | 策略決策 |
|------|----------|
| **目標市場** | 台灣為首發市場，後續擴展至亞太繁中地區 |
| **差異化定位** | 專注食物戒除場景 + 正向心理學 + 繁中本地化 |
| **商業模式** | Freemium 模式，Premium 訂閱制 |
| **定價策略** | 台灣 NT$99/月 或 NT$799/年 |
| **GTM 策略** | 內容行銷 + 社群經營 + KOL 合作 |
| **北極星指標** | 週活躍打卡用戶數 (WACU) |

### 財務預測摘要

| 階段 | 時間 | 預估 MAU | 預估 MRR |
|------|------|----------|----------|
| MVP | 月 3 | 500 | NT$15,000 |
| V1.1 | 月 6 | 2,000 | NT$60,000 |
| V1.2 | 月 12 | 8,000 | NT$300,000 |

#### TAM / SAM / SOM 估算

```
TAM (Total Addressable Market) - 全球可觸及市場
========================================
全球習慣追蹤 App 市場（食物/健康相關細分）
估算：USD 1.9B x 20%（食物相關）= USD 380M

SAM (Serviceable Addressable Market) - 可服務市場
========================================
亞太繁體中文市場（台灣、香港、部分海外華人）
估算：USD 380M x 5%（繁中市場佔比）= USD 19M
約等於 NT$600M / 年

SOM (Serviceable Obtainable Market) - 可獲得市場
========================================
首年目標市場份額（台灣市場）
估算：NT$600M x 1%（首年市佔率）= NT$6M / 年
```

#### 市場機會分析

| 驅動因素 | 說明 |

## 商業模式設計

### 1. 免費功能 vs Premium 功能劃分

#### 功能分層設計原則

| 原則 | 說明 |
|------|------|
| **核心價值免費** | 讓用戶免費體驗產品核心功能，建立使用習慣 |
| **進階需求付費** | 深度使用者願意為進階功能付費 |
| **軟性限制** | 避免硬性 paywall，採用功能限制而非完全封鎖 |
| **價值驅動升級** | 讓用戶因為「想要更多」而非「被迫」升級 |

## 北極星指標與 KPI 體系

### 1. 北極星指標定義

#### 北極星指標：週活躍打卡用戶數 (WACU)

| 元素 | 定義 |
|------|------|
| **指標名稱** | Weekly Active Checkin Users (WACU) |
| **定義** | 過去 7 天內至少打卡 1 次的獨立用戶數 |
| **計算公式** | COUNT(DISTINCT user_id) WHERE checkin_date >= NOW() - 7 days |

#### 為什麼選擇 WACU？

| 原則 | 說明 |
|------|------|
| **反映核心價值** | 打卡是 Cheese-Thief 的核心使用行為 |
| **領先指標** | WACU 與留存率、付費轉換率高度相關 |
| **可行動** | 產品優化可直接影響此指標 |
| **客觀可測量** | 無歧義、易於追蹤 |

#### 北極星指標目標

| 階段 | 時間 | WACU 目標 | 成長策略 |
|------|------|-----------|----------|
| MVP | 月 1-3 | 100 → 300 | 用戶獲取、Onboarding 優化 |
| V1.1 | 月 4-6 | 400 → 1,200 | 功能完善、提醒優化 |
| V1.2 | 月 7-12 | 1,500 → 5,000 | 社群功能、病毒式傳播 |

### 2. 緩解策略

#### 風險 1: 差異化定位不被市場認可

| 面向 | 緩解策略 |
|------|----------|
| **預防** | 上線前進行用戶訪談，驗證價值主張 |
| **監測** | 追蹤 NPS 與用戶反饋，辨識價值認知 |
| **應對** | 準備 Pivot 方案：擴展至更廣的「習慣戒除」場景 |
| **指標** | NPS < 30 或「食物專注」功能使用率 < 50% 觸發警報 |

#### 風險 2: 用戶留存率低於預期

| 面向 | 緩解策略 |
|------|----------|
| **預防** | 強化 Onboarding 體驗，確保用戶快速獲得價值 |
| **監測** | 每日追蹤 D1/D7/D30 留存率 |

| **應對-2** | 提早引入社群功能，增加社交黏性 |
| **應對-3** | 遊戲化元素強化（連續打卡獎勵） |
| **指標** | D7 留存率 < 25% 觸發警報 |

#### 風險 3: 免費轉付費率不足

| 面向 | 緩解策略 |
|------|----------|
| **預防** | 設計清晰的 Premium 價值溝通 |
| **監測** | 追蹤 Paywall 顯示、點擊、轉換漏斗 |
| **應對-1** | 優化 Paywall 設計與文案 |
| **應對-2** | 提供限時優惠或試用期 |
| **應對-3** | 強化 Premium 專屬功能價值 |
| **指標** | 免費轉付費率 < 2% 觸發警報 |

#### 風險 4: 技術債累積

| 面向 | 緩解策略 |
|------|----------|

| **應對** | 每個 Sprint 預留 20% 時間處理技術債 |
| **指標** | 測試覆蓋率 < 70% 或 Bug 修復時間 > 3 天觸發警報 |

#### 風險 5: 離線同步機制複雜

| 面向 | 緩解策略 |

| **監測** | 追蹤同步失敗率、用戶反饋 |

#### 風險 6: 競品快速迭代

| 面向 | 緩解策略 |

### C. KPI 計算公式

```sql
-- 週活躍打卡用戶數 (WACU)
SELECT COUNT(DISTINCT user_id) as WACU
FROM daily_checkins
WHERE checkin_date >= CURRENT_DATE - INTERVAL '7 days';

-- D7 留存率
SELECT
  COUNT(DISTINCT CASE WHEN day_7_active THEN user_id END) * 100.0 /
  COUNT(DISTINCT user_id) as d7_retention_rate
FROM (
  SELECT
    u.user_id,
    EXISTS (
      SELECT 1 FROM daily_checkins dc
      WHERE dc.user_id = u.user_id
      AND dc.checkin_date = u.signup_date + INTERVAL '7 days'
    ) as day_7_active
  FROM users u
  WHERE u.signup_date = CURRENT_DATE - INTERVAL '7 days'
) cohort;

-- 免費轉付費率
SELECT
  COUNT(DISTINCT CASE WHEN subscription_status = 'premium' THEN user_id END) * 100.0 /
  COUNT(DISTINCT user_id) as conversion_rate
FROM users
WHERE signup_date <= CURRENT_DATE - INTERVAL '30 days';
```
