# 一夜終極狼人殺 (One Night Ultimate Werewolf) - 產品策略分析

> **產出者**: Product Strategist
> **日期**: 2026-03-12
> **版本**: v1.0

---

## 1. 市場定位分析

### 1.1 為什麼選擇一夜終極狼人殺

一夜終極狼人殺（One Night Ultimate Werewolf, ONUW）是 Bezier Games 出版的極簡社交推理桌遊，在 BoardGameGeek 上長期位居派對遊戲前 20 名。阿瓦隆產品策略中已將其列為「第三款遊戲最推薦候選」，原因如下：

**遊戲本身的市場優勢：**

- **極短遊戲時間**：單局僅 10-15 分鐘，與阿瓦隆（30-45 分鐘）形成完美互補
- **極低學習門檻**：只有一個夜晚 + 一個白天，規則解說 3 分鐘內完成
- **高重玩性**：每局角色配置可變，角色互動產生大量不可預測情境
- **知名度極高**：全球累計銷量超過百萬套，在台灣桌遊圈是派對聚會的標準配備
- **彈性人數**：3-10 人皆可遊玩，特別適合 5-7 人的小型聚會
- **聚會暖場首選**：短局制讓它成為聚會開場、等人到齊時的最佳選擇

**對產品矩陣的戰略價值：**

```
遊戲深度梯度：

輕度暖場 ──一夜狼人 (10 min)──> 中度推理 ──阿瓦隆 (30-45 min)──> 深度策略（未來）
  (第三款)                          (第二款)                         (血染鐘樓)

聚會場景覆蓋：

等人到齊時 → 一夜狼人
正式開局後 → 阿瓦隆
深度場合   → 未來擴展
```

新增一夜狼人後，產品將覆蓋桌遊聚會中 80% 以上的時段需求。

### 1.2 遊戲核心機制

一夜狼人與傳統狼人殺有根本性區別：

| 維度 | 傳統狼人殺 | 一夜終極狼人殺 |
|------|-----------|---------------|
| 遊戲長度 | 30-60 分鐘 | 10-15 分鐘 |
| 夜晚次數 | 多個夜晚循環 | 僅一個夜晚 |
| 淘汰機制 | 有（玩家逐輪淘汰） | 無（所有人玩到最後） |
| 角色變動 | 角色固定 | 角色可能在夜晚被交換 |
| 核心張力 | 長線推理、多輪博弈 | 短時間高強度推理、身份不確定性 |
| 桌面額外卡牌 | 無 | 3 張中央牌（無人持有） |
| 結束方式 | 某方全滅 | 一次投票，得票最高者死亡 |

**為什麼語音主持對一夜狼人至關重要：**

一夜狼人的夜晚階段有嚴格的角色行動順序，每個角色需要在特定時機睜眼執行動作，順序錯誤會直接破壞遊戲。官方本身就出了 App 來處理這個問題，證明語音主持是該遊戲的剛需。

### 1.3 目標用戶群體分析

| 維度 | 奶酪大盜用戶 | 阿瓦隆用戶 | 一夜狼人用戶 | 重疊度 |
|------|-------------|-----------|-------------|--------|
| 人數需求 | 4-8 人 | 5-10 人 | 3-10 人 | 極高 |
| 遊戲深度 | 輕度推理 | 中度策略 | 輕中度推理 | 互補 |
| 典型場景 | 家庭/輕鬆聚會 | 桌遊社/深度聚會 | 派對暖場/快速對局 | 互補 |
| 年齡層 | 12+ | 14+ | 10+ | 高度重疊 |
| 遊戲時長 | 20-30 分鐘 | 30-45 分鐘 | 10-15 分鐘 | 形成梯度 |
| 主持需求 | 低 | 高 | 高 | 高度匹配 |

**新增觸及的用戶群：**

1. **派對常客** -- 一夜狼人是 KTV、聚餐後最常被拿出來的桌遊，頻次極高
2. **桌遊入門者** -- 一夜狼人的低門檻特性吸引大量非桌遊核心玩家
3. **時間碎片化玩家** -- 只有 15 分鐘也能開一局，填補等待空檔
4. **多次連開玩家** -- 一夜狼人的短局制讓人習慣連續開 3-5 局

### 1.4 競品分析

| 競品 | 類型 | 語音主持 | 離線 | 多遊戲 | 免費 | 缺點 |
|------|------|---------|------|--------|------|------|
| One Night 官方 App | 官方配套 App | 有（高品質錄製語音） | 是 | 僅 One Night 系列 | 是 | 僅支援英文等語言，無繁體中文語音；僅限 One Night 系列；無遊戲結算功能 |
| 一夜終極狼人小幫手（網頁版） | 社群工具 | 有（基礎 TTS） | 部分 | 否 | 是 | 介面簡陋；功能有限；缺乏白天計時與結算流程 |
| 狼人殺 MC App | 主持工具 | 有（粵語） | 是 | 僅狼人殺 | 是 | 粵語為主；僅傳統狼人殺，不支援一夜模式 |
| 終極狼人殺 App | 線上對戰 | 有 | 否 | 部分 | 免費+內購 | 線上多人模式，非面對面場景；需要每人一台設備 |
| onenightwerewolf.com | 線上對戰 | 否 | 否 | 否 | 是 | 線上版本，需每人一台設備 |

**競品缺口分析：**

```
                   有繁體中文語音
                       |
  一夜狼人小幫手 (弱)  |  BoardGame Voice Host
                       |  (目標位置)
  ─────────────────────┼─────────────────────→ 完整遊戲流程
                       |                        (夜晚+白天+結算)
  One Night 官方 App   |  (市場空白)
  (無中文)             |
                       |
                   無繁體中文語音
```

**核心機會：**

1. **繁體中文語音主持的空白** -- 官方 App 不支援繁體中文語音，這是台灣市場的痛點
2. **完整遊戲流程覆蓋** -- 現有工具多只做夜晚主持，缺乏白天計時與投票結算
3. **多遊戲整合** -- 一夜狼人打完接著玩阿瓦隆，不需切換 App
4. **純離線設計** -- 不需網路，適合各種聚會場景（包括網路不穩的場所）

## 2. 核心價值主張

### 2.1 產品定位

**一句話定位**：讓手機成為一夜狼人的完美主持人 -- 繁體中文語音引導夜晚流程，白天自動計時，投票即時結算。

**解決的核心痛點**：

| 痛點 | 當前解法 | 我們的解法 |
|------|---------|-----------|
| 夜晚流程需要主持人背台詞 | 找一個人讀稿 / 用官方英文 App | 繁體中文 TTS 自動播報 |
| 主持人不能參與遊戲 | 犧牲一個玩家當主持 | 所有人都能玩 |
| 角色行動順序容易搞錯 | 主持人死記硬背 | App 自動按正確順序引導 |
| 白天討論沒有時限 | 憑感覺喊停 | 可調整的倒數計時器 |
| 投票結果需要人工計算 | 數手指 | App 即時結算並宣布結果 |
| 想換角色配置很麻煩 | 重新翻規則書 | 一鍵切換預設配置 / 自由選角 |

### 2.2 與官方 App 的差異化

官方 One Night App 是最直接的競品，差異化策略如下：

| 維度 | 官方 One Night App | BoardGame Voice Host |
|------|-------------------|---------------------|
| 語言 | 英文、希伯來文、俄文等（無繁中） | 繁體中文（主要）+ 可擴展 |
| 遊戲範圍 | 僅 One Night 系列 | 一夜狼人 + 阿瓦隆 + 更多 |
| 白天流程 | 僅提供計時器 | 計時 + 投票 + 結算 |
| 離線能力 | 有 | 有（PWA 完全離線） |
| 安裝方式 | 需下載 Native App | PWA 免安裝，掃碼即用 |
| 自訂性 | 中等 | 高（語速、計時、角色配置） |

**我們不做的事（避免與官方正面競爭）**：
- 不做英文語音（官方 App 已做到極致）
- 不做付費解鎖角色包（我們定位為免費工具）
- 不做每人一台設備的線上模式

### 2.3 App 的主持範圍（重要邊界定義）

根據專案定位「App 只負責語音主持夜晚流程 + 白天計時/結算，不做數位角色分配（用實體卡牌）」，明確定義：

**App 負責的：**
- 語音播報夜晚階段各角色的行動指令（按正確順序）
- 夜晚階段各角色的行動時間控制（含音效提示）
- 白天討論計時
- 投票倒數與結果統計
- 遊戲結果宣布

**App 不負責的（由實體卡牌 + 玩家自行處理）：**
- 角色卡牌的發放（用實體卡牌面朝下分配）
- 夜晚角色行動的實際執行（玩家自己翻牌、交換牌等）
- 中央 3 張牌的管理（實體卡牌放桌面中央）

這意味著 App 的定位是「語音計時器 + 流程引導」，而非「完整數位化遊戲」。玩家仍需持有實體遊戲卡牌。

## 3. 功能優先級 (MoSCoW)

### Must Have（核心功能 -- 無此不可）

| 功能 | 說明 | 理由 |
|------|------|------|
| 角色選擇介面 | 根據人數選擇本局使用的角色（角色數 = 玩家數 + 3 中央牌） | 遊戲設定的基礎，決定夜晚流程 |
| 夜晚語音主持 | TTS 按正確順序引導每個角色的行動（睜眼、執行動作、閉眼） | 替代真人主持的核心功能 |
| 角色行動順序引擎 | 根據選擇的角色自動排列正確的夜晚行動順序 | 順序錯誤會破壞遊戲，必須正確 |
| 角色行動計時 | 每個角色行動的固定時長（含開始/結束提示音） | 控制節奏，防止行動時間過長暴露身份 |
| 白天討論計時器 | 可調整的白天討論倒數計時（預設 5 分鐘） | 防止討論無限拖延 |
| 投票機制 | 倒數 3-2-1 同時指人投票的語音提示 | 一夜狼人的標準投票方式 |
| 基礎角色集 | 狼人、預言家、強盜、搗蛋鬼、失眠者、獵人、村民、皮匠、爪牙 | 基礎版核心角色 |
| 遊戲結果判定 | 根據投票結果 + 角色身份判定勝負（好人陣營 / 狼人陣營 / 皮匠） | 遊戲結束的必要功能 |

### Should Have（增強體驗 -- 顯著提升品質）

| 功能 | 說明 | 理由 |
|------|------|------|
| 人數推薦配置 | 根據玩家人數推薦官方建議的角色組合 | 降低新手配置門檻 |
| 音效系統 | 夜晚氛圍音效、角色行動提示音、投票倒數音效 | 提升沉浸感和儀式感 |
| 語速調整 | 允許調整 TTS 播報速度（新手慢 / 老手快） | 適應不同玩家群體 |
| 每局角色行動時長調整 | 允許調整每個角色的行動時間 | 新手需要更多時間思考 |
| 遊戲歷史記錄 | 本地 IndexedDB 記錄每局的角色配置、投票結果、勝負 | 覆盤和統計趣味 |
| 角色說明卡 | 長按角色可查看該角色的能力說明 | 新手隨時查閱規則 |
| 夜晚背景音樂 | 持續播放的低沉背景音樂，掩蓋實際動作聲音 | 防止透過聲音判斷誰在行動 |

### Could Have（錦上添花 -- 有則更好）

| 功能 | 說明 | 理由 |
|------|------|------|
| Daybreak 擴展角色 | 加入破曉擴展包的角色（暴民、超自然迷、村白癡等） | 擴展遊戲深度，覆蓋進階玩家 |
| 自訂角色行動腳本 | 允許修改角色的語音播報文字 | 進階自訂需求 |
| 多種語音風格 | 不同 TTS 聲音選擇（嚴肅 / 搞怪 / 恐怖） | 氣氛差異化 |
| 角色配置收藏 | 將常用的角色組合儲存為預設 | 頻繁玩家的便利功能 |
| 快速再開一局 | 一鍵使用相同配置開始新遊戲 | 連續對局的便利性 |
| 統計數據 | 各角色配置的使用頻率、各人數的場次分佈 | 數據趣味性 |
| 規則教學模式 | 互動式角色能力教學 | 降低新手門檻 |

### Won't Have（不做 -- 明確排除）

| 功能 | 理由 |
|------|------|
| 數位角色分配 | 專案定位明確使用實體卡牌，App 不取代實體遊戲 |
| 夜晚角色行動的數位化操作 | 交換牌、查看牌等動作由玩家用實體卡牌執行 |
| 線上多設備模式 | 違反離線優先、一台設備的設計原則 |
| AI 玩家 | 產品是主持工具，不是遊戲平台 |
| 帳號系統 / 雲端同步 | 離線優先、隱私優先 |
| Vampire / Alien / Super Villains 擴展 | V1 範圍過大，先做好基礎版 + Daybreak |
| 社交功能 | 不符合產品定位 |

### 4.3 開發成本評估

| 維度 | 阿瓦隆 | 一夜狼人 | 差異說明 |
|------|--------|---------|---------|

## 5. KPI 定義

### 5.1 北極星指標

**一夜狼人每週完成遊戲場次 (ONUW Weekly Completed Games)**

定義：過去 7 天內，一夜狼人遊戲從夜晚開始到投票結算完成的總場次。

目標：
| 時間 | 目標 | 說明 |
|------|------|------|
| 上線後 2 週 | 100 場/週 | 一夜狼人的短局制預期場次高於阿瓦隆 |
| 上線後 1 個月 | 400 場/週 | 社群推廣 + 現有用戶轉化 |
| 上線後 3 個月 | 1,000 場/週 | 穩定成長，含自然裂變 |

注：一夜狼人因單局僅 10-15 分鐘，預期每次聚會開 3-5 局，場次目標應高於阿瓦隆。

### 5.2 輔助指標

| 指標 | 定義 | 目標 | 追蹤方式 |
|------|------|------|---------|
| 遊戲完成率 | 開始夜晚 vs 完成投票結算的比例 | > 85% | IndexedDB 記錄 |
| 平均每次聚會場次 | 同一設備同一天內的遊戲場次 | > 3 場 | IndexedDB 記錄 |
| 平均遊戲時長 | 從夜晚開始到結算完成的時間 | 8-15 分鐘 | IndexedDB 記錄 |
| 角色配置多樣性 | 使用不同角色組合的比例 | > 4 種配置 | IndexedDB 記錄 |
| 跨遊戲使用率 | 同一設備同時使用一夜狼人 + 其他遊戲 | > 30% | IndexedDB 記錄 |
| 重複使用率 | 同一設備 7 天內再次使用一夜狼人 | > 45% | IndexedDB 記錄 |
| 平均玩家人數 | 每局的角色數 - 3（中央牌） | > 5 人 | IndexedDB 記錄 |

### 5.3 健康指標

| 指標 | 定義 | 警戒線 | 說明 |
|------|------|--------|------|
| 遊戲中途放棄率 | 開始但未完成的比例 | < 15% | 若過高可能表示流程有問題 |
| 投票階段放棄率 | 進入白天但未完成投票 | < 10% | 若過高可能表示投票 UX 有問題 |
| TTS 錯誤率 | 語音播報失敗的比例 | < 2% | 若過高需修正 TTS 相容性 |
| 平均角色行動時間使用率 | 玩家實際使用的行動時間占分配時間的比例 | 50-80% | 過低表示時間太長，過高表示時間不夠 |

### 5.4 建議追蹤的本地數據（IndexedDB）

```typescript
interface OneNightGameRecord {
```

## 7. 風險評估

### 7.1 技術風險

| 風險 | 嚴重度 | 可能性 | 描述 | 緩解方案 |
|------|--------|--------|------|---------|
| TTS 角色順序錯誤 | 高 | 低 | 角色行動順序硬編碼錯誤導致遊戲破壞 | 嚴格按官方順序實作；建立完整的角色順序測試 |
| TTS 連續播報中斷 | 高 | 中 | 夜晚階段多段語音連續播報，中途被系統中斷 | 沿用已驗證的 TTS 方案；每段語音之間加入靜音 buffer |
| iOS 鎖屏中斷 | 高 | 高 | 夜晚階段手機自動鎖屏導致語音停止 | 使用 Wake Lock API；提示用戶設定螢幕常亮 |
| 行動計時精準度 | 中 | 中 | 不同設備的計時精度差異 | 使用 performance.now() 而非 setInterval |

### 7.2 用戶體驗風險

| 風險 | 嚴重度 | 可能性 | 描述 | 緩解方案 |
|------|--------|--------|------|---------|
| 與官方 App 的比較 | 高 | 高 | 用戶可能與官方 App 的錄製語音品質做比較 | 強調繁體中文語音的獨特價值；持續優化 TTS 參數 |
| 行動時間過短 / 過長 | 中 | 高 | 不同玩家對行動時間需求不同 | 提供可調整的行動時間設定 |
| 角色能力理解錯誤 | 中 | 高 | 新手不理解角色能力導致遊戲混亂 | 遊戲開始前提供角色說明；語音播報時簡要提示行動內容 |
| 實體卡牌與 App 不匹配 | 中 | 中 | 玩家使用的實體卡牌版本與 App 角色名稱不一致 | 使用最通用的角色中文名稱；提供角色名稱對照 |
| 投票結果爭議 | 低 | 中 | 平票或投票規則理解不一致 | 明確顯示投票規則（平票規則等） |

### 7.3 市場風險

| 風險 | 嚴重度 | 可能性 | 描述 | 緩解方案 |

## 9. 關鍵建議總結

### 為什麼現在做一夜狼人

1. **用戶需求已驗證** -- 官方 App 的存在本身就證明語音主持是剛需，而繁中市場是空白
2. **開發成本低** -- 單輪遊戲邏輯簡單，大量基礎設施可從阿瓦隆複用
3. **產品矩陣互補** -- 填補「快速暖場遊戲」的空缺，形成完整的聚會工具鏈
4. **用戶獲取成本低** -- 透過現有用戶交叉推廣，邊際獲客成本趨近於零
5. **場次潛力高** -- 短局制意味著單次聚會貢獻 3-5 場遊戲，大幅提升北極星指標

### 最關鍵的成功因素

1. **角色行動順序必須正確** -- 這是遊戲的基礎，任何順序錯誤都會導致遊戲崩壞
2. **夜晚語音播報的節奏感** -- 每個角色的行動提示之間需要適當的停頓和音效
3. **繁體中文語音的自然度** -- TTS 的角色名稱和行動描述需要聽起來自然
4. **與實體卡牌的無縫配合** -- App 的指令必須與玩家手上的實體操作完美對應

### 不要做的事

1. 不要做數位角色分配 -- 忠於「實體卡牌 + 語音主持」的定位
2. 不要第一版就加 Daybreak 擴展 -- 先做好基礎版 9 角色
3. 不要試圖取代官方 App -- 專注繁中市場和多遊戲整合的差異化
4. 不要過度設計 UI -- 一夜狼人的 App 介面應該極簡，重點在語音
