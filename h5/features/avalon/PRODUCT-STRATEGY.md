# 阿瓦隆 (Avalon) 遊戲功能 - 產品策略分析

> **產出者**: Product Strategist
> **日期**: 2026-03-11
> **版本**: v1.0

## 1. 市場定位分析

### 1.1 阿瓦隆在桌遊市場的受歡迎程度

阿瓦隆（The Resistance: Avalon）是全球最受歡迎的社交推理桌遊之一，在 BoardGameGeek 上長期位居派對遊戲前列。其核心優勢在於：

- **高重玩性**：角色配置變化多，每局體驗不同
- **零消耗品**：不需要額外卡牌消耗（對比狼人殺需要淘汰標記）
- **學習曲線適中**：比血染鐘樓簡單，比誰是臥底有深度
- **最佳人數覆蓋廣**：5-10 人皆可，特別適合 7-8 人的社團聚會

**台灣市場特點**：
- 桌遊店幾乎必備的遊戲之一
- 大學桌遊社的入門社交推理遊戲
- PTT 桌遊版、Dcard 桌遊板經常討論的遊戲
- 阿瓦隆相關教學影片在 YouTube 中文圈有穩定流量

### 1.2 目標用戶群體分析

| 維度 | 奶酪大盜用戶 | 阿瓦隆用戶 | 重疊度 |
|------|-------------|-----------|--------|
| 人數需求 | 4-8 人 | 5-10 人 | 高（5-8 人重疊） |
| 遊戲深度 | 輕度推理 | 中度策略+社交推理 | 互補 |
| 典型場景 | 家庭/輕鬆聚會 | 桌遊社/深度玩家聚會 | 部分重疊 |
| 年齡層 | 12+ | 14+ | 高度重疊 |
| 遊戲時長 | 20-30 分鐘 | 30-45 分鐘 | 相近 |

**用戶擴展效果**：

奶酪大盜定位為「輕量入門」，阿瓦隆定位為「經典進階」。兩者形成用戶梯度：

```
輕度玩家 ──奶酪大盜──> 中度玩家 ──阿瓦隆──> 深度玩家（未來：血染鐘樓）
```

新增阿瓦隆預計可觸及以下新用戶群：

1. **大學桌遊社成員** — 阿瓦隆是社團聚會最常見遊戲之一
2. **桌遊店常客** — 店家經常需要主持阿瓦隆，語音主持可降低人力
3. **老練桌遊玩家** — 奶酪大盜對他們偏輕，阿瓦隆才是剛需

### 1.3 競品分析

| 競品 | 類型 | 語音主持 | 離線 | 多遊戲 | 缺點 |
|------|------|---------|------|--------|------|
| 阿瓦隆小幫手（網頁版） | 簡易工具 | 有（基礎） | 部分 | 否 | 介面簡陋、無流程控制、無任務追蹤 |
| Wi-Fi Avalon | 線上對戰 | 否 | 否 | 否 | 需每人一台設備、需網路 |
| AvalonGO | 角色指引 | 否 | 是 | 否 | 僅角色分配、無完整主持 |
| Avalon Companion 類 App | 輔助工具 | 部分 | 是 | 否 | 功能零散、無語音流程 |

**競品缺口（我們的機會）**：

目前市場上**沒有**一款 App 能做到：
1. 完整的阿瓦隆語音自動主持（從角色分配到任務投票的全流程）
2. 離線運作、一台設備即可
3. 同時支援多款桌遊

這正是 BoardGame Voice Host 的核心差異化。阿瓦隆小幫手（z358z358.github.io）是最接近的競品，但僅提供基礎語音播報，缺乏完整的遊戲流程管理（組隊、投票、任務執行、刺殺階段等）。

## 2. 功能優先級建議 (MoSCoW)

### Must Have（核心玩法 - 無此不可）

| 功能 | 說明 | 理由 |
|------|------|------|
| 角色配置選擇 | 根據人數推薦角色配置（梅林、派西維爾、莫甘娜、莫德雷德、奧伯倫等） | 遊戲核心，不同角色組合決定遊戲深度 |
| 夜晚階段語音主持 | TTS 自動引導：壞人互認 > 梅林睜眼 > 派西維爾睜眼 | 替代真人主持的核心功能 |
| 隊長組隊機制 | 顯示當前隊長、任務所需人數、允許選擇隊員 | 遊戲核心流程 |
| 全體投票（同意/反對） | 所有玩家對組隊方案投票 | 遊戲核心流程 |
| 任務執行（成功/失敗） | 被選中的隊員秘密投出成功或失敗牌 | 遊戲核心流程 |
| 任務結果記錄與顯示 | 顯示 5 回合任務進度（3 成功好人贏 / 3 失敗壞人贏） | 遊戲狀態一目了然 |
| 刺殺階段 | 好人完成 3 次任務後，壞人嘗試刺殺梅林 | 阿瓦隆最經典的翻盤機制 |
| 投票追蹤器 | 記錄連續被否決的組隊次數（第 5 次強制通過） | 規則必要 |
| 各人數任務配置表 | 5-10 人對應的每回合任務人數 | 規則基礎資料 |

### Should Have（增強體驗 - 顯著提升品質）

| 功能 | 說明 | 理由 |
|------|------|------|
| 湖中女神（Lady of the Lake） | 可選角色，每回合後可查驗一名玩家陣營 | 官方擴展規則，增加遊戲深度 |
| 任務歷史回顧 | 查看每回合的組隊方案、投票結果、任務結果 | 幫助玩家推理分析 |
| 音效與背景音樂 | 夜晚氛圍音效、投票倒數音效、任務成功/失敗音效 | 提升沉浸感 |
| 遊戲歷史記錄 | 本地 IndexedDB 儲存每局遊戲結果 | 覆盤分析、成就感來源 |
| TTS 語速/語調設定 | 允許調整語音播報速度 | 不同場景需求不同（新手需慢、老手可快） |
| 玩家座位順序管理 | 視覺化顯示座位順序、隊長輪轉 | 降低遊戲管理混亂 |

### Could Have（錦上添花 - 有則更好）

| 功能 | 說明 | 理由 |
|------|------|------|
| 自訂角色配置 | 允許自由增減角色、調整陣營平衡 | 進階玩家需求 |
| 多語音風格 | 不同 TTS 聲音選擇（嚴肅/歡樂/中世紀風格） | 氣氛差異化 |
| 規則教學模式 | 互動式規則教學，適合新手 | 降低學習門檻 |
| 統計數據 | 各角色勝率、各人數配置勝率 | 數據驅動的趣味性 |
| 夜晚階段動畫 | 閉眼/睜眼的視覺動畫效果 | 提升儀式感 |
| 抵抗組織模式 | 基礎版無特殊角色的抵抗組織玩法 | 低成本擴展遊戲變體 |

### Won't Have（不做 - 明確排除）

| 功能 | 理由 |
|------|------|
| 線上多設備對戰 | 違反離線優先設計原則，且改變了「面對面一台設備」的核心定位 |
| AI 玩家 / Bot | 本產品是「主持工具」而非「遊戲平台」，玩家必須是真人 |
| 即時語音通訊 | 面對面場景不需要，且增加技術複雜度 |
| 帳號系統 / 雲端同步 | 離線優先、隱私優先的設計原則 |
| 自訂遊戲模板編輯器 | V1 階段過於複雜，阿瓦隆規則固定，不需要 |
| 社交功能（好友、排行） | 不符合產品定位 |

## 3. 成功指標 (KPIs)

### 3.1 北極星指標

**阿瓦隆每週完成遊戲場次 (Avalon Weekly Completed Games)**

定義：過去 7 天內，阿瓦隆遊戲從開始到結束（任一方獲勝或刺殺完成）的總場次。

目標：
| 時間 | 目標 | 說明 |
|------|------|------|
| 上線後 2 週 | 50 場/週 | 早期採用者驗證 |
| 上線後 1 個月 | 200 場/週 | 社群推廣效果 |
| 上線後 3 個月 | 500 場/週 | 穩定成長 |

### 3.2 輔助指標

| 指標 | 定義 | 目標 | 追蹤方式 |
|------|------|------|---------|
| 遊戲完成率 | 開始遊戲 vs 完成遊戲的比例 | > 75% | IndexedDB 記錄 |
| 平均遊戲時長 | 每局遊戲的平均時間 | 25-40 分鐘 | IndexedDB 記錄 |
| 角色配置多樣性 | 使用不同角色配置的比例 | > 3 種配置 | IndexedDB 記錄 |
| 重複使用率 | 同一設備 7 天內再次玩阿瓦隆 | > 40% | IndexedDB 記錄 |
| 遊戲切換率 | 同一設備同時玩奶酪大盜+阿瓦隆 | > 20% | IndexedDB 記錄 |
| 平均玩家人數 | 每局的平均參與人數 | > 6 人 | IndexedDB 記錄 |

### 3.3 建議追蹤的本地數據（IndexedDB）

```typescript
interface AvalonGameRecord {
  id: string
  startedAt: string                    // 遊戲開始時間
  endedAt: string | null               // 遊戲結束時間
  completed: boolean                   // 是否完成
  playerCount: number                  // 玩家人數
  roleConfig: string[]                 // 使用的角色列表
  rounds: number                       // 進行了幾回合
  goodWins: boolean | null             // 好人是否獲勝
  assassinationUsed: boolean           // 是否進入刺殺階段
  assassinationSuccess: boolean | null // 刺殺是否成功
  votingRejections: number             // 總被否決次數
  deviceId: string                     // 匿名設備識別（用於重複使用率計算）
}
```

**隱私說明**：所有數據僅存在本地 IndexedDB，不傳送至任何伺服器，符合專案的零數據收集原則。統計數據僅在設備本地的「歷史記錄」頁面展示。

## 4. 推廣策略

### 4.1 利用阿瓦隆知名度帶動 App 使用

**核心策略：「不需要主持人的阿瓦隆」**

阿瓦隆最大的使用痛點是**需要一個熟悉規則的主持人引導夜晚階段**。許多桌遊聚會因為「沒人願意當主持人」而放棄玩阿瓦隆。這是一個明確且普遍的痛點。

### 4.2 社群推廣建議

#### Phase 1：種子期（上線前 2 週 - 上線後 2 週）

| 管道 | 行動 | 預期效果 |
|------|------|---------|
| PTT BG_Tabletop 板 | 發布「阿瓦隆語音主持工具分享」心得文 | 精準觸及桌遊核心玩家 |
| Dcard 桌遊板 | 「再也不用搶著當主持人了」體驗文 | 觸及大學生桌遊群體 |
| 桌遊 Facebook 社團 | 「桌遊人的小工具」分享貼文（3-5 個大社團） | 觸及 10,000+ 桌遊愛好者 |
| cheese-thief.pages.dev | 首頁阿瓦隆卡片從「即將推出」變為「新遊戲」 | 轉化現有用戶 |

#### Phase 2：擴散期（上線後 2-8 週）

| 管道 | 行動 | 預期效果 |
|------|------|---------|
| YouTube 桌遊頻道 | 聯繫 2-3 個中小型桌遊 YouTuber 試用 | 影片形式展示效果最佳 |
| 桌遊店合作 | 提供 5 家桌遊店試用，收集反饋 | 線下場景驗證 + 口碑 |
| LINE 社群 | 在桌遊相關 LINE 群組分享 | 即時觸及聚會組織者 |

#### Phase 3：口碑期（上線後 2 個月+）

| 管道 | 行動 | 預期效果 |
|------|------|---------|
| App 內分享 | 遊戲結束後「分享給朋友」按鈕 | 自然裂變 |
| SEO | 優化「阿瓦隆線上工具」「阿瓦隆主持」等關鍵字 | 長尾搜尋流量 |
| UGC 鼓勵 | 鼓勵用戶分享遊戲截圖/心得 | 社群自然成長 |

### 4.3 推廣文案建議

- **核心 Slogan**：「放下手機當主持人，讓 App 來」
- **社群標題**：「阿瓦隆再也不用搶誰當主持了 — 免費語音主持工具」
- **功能亮點**：「一鍵開局、語音引導夜晚流程、支援 5-10 人、完全離線免費」

## 5. 風險評估

### 5.1 技術風險

| 風險 | 嚴重度 | 可能性 | 描述 | 緩解方案 |
|------|--------|--------|------|---------|
| TTS 語音品質不一致 | 中 | 高 | 不同設備/瀏覽器的 Web Speech API 語音品質差異大 | 提供語音測試功能，允許用戶選擇最佳語音；針對常見設備做語音參數微調 |
| iOS Safari TTS 限制 | 高 | 高 | iOS 要求用戶互動才能播放語音，中途可能被系統中斷 | 沿用奶酪大盜的 unlockTts() 方案；每個階段開始前加入用戶確認點擊 |
| 夜晚階段計時問題 | 中 | 中 | 手機鎖屏或切換 App 可能導致計時器暫停 | 使用 Page Visibility API 偵測，提醒用戶保持螢幕常亮；提供「保持螢幕開啟」選項 |
| IndexedDB 儲存限制 | 低 | 低 | 大量遊戲歷史可能超出瀏覽器配額 | 設定最大記錄數（如 100 局），自動清理最舊記錄 |

### 5.2 用戶體驗風險

| 風險 | 嚴重度 | 可能性 | 描述 | 緩解方案 |
|------|--------|--------|------|---------|
| 夜晚階段偷看問題 | 高 | 高 | 面對面場景中，玩家可能偷看螢幕得知他人角色 | 夜晚階段最小化螢幕資訊；角色確認採用「傳閱手機」模式，確認後立即隱藏 |
| 規則理解門檻 | 中 | 中 | 新手不了解阿瓦隆規則，語音主持無法完全取代教學 | Must Have 中不含教學模式，但在角色分配前提供簡要規則摘要文字 |
| 投票操作繁瑣 | 中 | 中 | 每回合多次投票（組隊投票 + 任務投票），操作步驟多 | 簡化投票 UI，大按鈕設計；投票流程用語音提示引導 |
| 遊戲中途退出 | 中 | 中 | 玩家中途離開導致遊戲無法繼續 | 提供「替換玩家」功能；遊戲狀態自動保存，可恢復 |
| 螢幕朝向問題 | 低 | 高 | 手機放桌上，不同位置的玩家看不清螢幕 | 關鍵資訊（任務結果、投票狀態）使用大字體；夜晚階段主要靠語音 |

### 5.3 遊戲平衡性風險

| 風險 | 嚴重度 | 可能性 | 描述 | 緩解方案 |
|------|--------|--------|------|---------|
| 角色配置不平衡 | 中 | 中 | 錯誤的角色組合可能導致某一方過強 | 提供官方推薦配置；非推薦配置顯示警告 |
| 隨機性爭議 | 低 | 中 | 角色分配的隨機性可能被質疑 | 使用 crypto.getRandomValues() 確保真隨機；顯示「角色已隨機分配」提示 |
| 特殊規則遺漏 | 中 | 低 | 某些邊緣規則（如第 4 回合雙失敗、7+ 人配置）遺漏 | 嚴格按官方規則實作；邊緣規則加入自動提醒 |

## 6. 功能路線圖建議

### 6.1 短期路線圖（3-6 個月）

```
2026 Q1 (現在)
├── 奶酪大盜 v1.0 ✅ 已上線
└── 阿瓦隆 v1.0 🔨 開發中
    ├── Must Have 功能
    └── 目標：2026 年 4 月上線

2026 Q2
├── 阿瓦隆 v1.1
│   ├── Should Have 功能（湖中女神、歷史記錄、音效）
│   └── 根據用戶反饋迭代
└── 第三款遊戲調研
```

### 6.2 第三款遊戲推薦

| 候選遊戲 | 推薦度 | 理由 |
|---------|--------|------|
| **一夜狼人 (One Night)** | **最推薦** | 極高知名度、快節奏（10分鐘/局）、與阿瓦隆用戶高度互補、主持流程清晰 |
| 狼人殺 (Werewolf) | 推薦 | 最高知名度、用戶基數最大、但流程複雜度高 |
| 誰是臥底 | 可考慮 | 簡單上手、但語音主持需求較低（核心是詞彙分配） |
| 血染鐘樓 | 長期 | 深度最高、但規則極複雜、開發成本大 |

**推薦優先順序**：一夜狼人 > 狼人殺 > 誰是臥底 > 血染鐘樓

**一夜狼人推薦理由**：
1. 遊戲時間極短（10 分鐘），與阿瓦隆（30-45 分鐘）形成完美互補
2. 只有一個夜晚，語音主持流程簡潔，開發成本低
3. 在台灣桌遊圈知名度極高
4. 可作為「聚會暖場」使用，引導用戶接著玩阿瓦隆

### 6.3 長期產品演進方向（6-12 個月）

```
2026 Q3
├── 一夜狼人 v1.0
├── 遊戲選擇首頁改版（3+ 遊戲的展示優化）
└── 全局設定頁面（語音、音效、語言偏好）

2026 Q4
├── 狼人殺 v1.0
├── PWA 安裝引導優化
└── 桌遊店合作方案（品牌化、大螢幕投放）

2027 Q1
├── 自訂遊戲模板系統（用戶可創建簡單的夜晚流程遊戲）
├── 多語言支援（簡體中文、英文）
└── 社群分享的遊戲模板
```

### 6.4 產品願景

```
短期 (V1.x)：最好的離線桌遊語音主持工具
      ↓
中期 (V2.x)：桌遊語音主持平台（用戶可創建模板）
      ↓
長期 (V3.x)：桌遊聚會的一站式工具（主持 + 計分 + 教學）
```

## 8. 關鍵建議總結

### 為什麼現在做阿瓦隆

1. **市場驗證低成本**：阿瓦隆規則成熟且固定，不需要設計遊戲，只需實作主持流程
2. **用戶需求明確**：「沒人當主持人」是真實且普遍的痛點
3. **競品空白**：目前沒有高品質的阿瓦隆語音主持離線工具
4. **產品矩陣互補**：與奶酪大盜形成輕度+中度的遊戲梯度
5. **開發可複用**：遊戲流程引擎（夜晚/白天/投票循環）可為後續狼人殺等遊戲複用

### 最關鍵的成功因素

1. **夜晚階段的語音主持體驗必須流暢** — 這是核心賣點，任何卡頓或錯誤都會嚴重影響遊戲體驗
2. **角色確認的隱私保護** — 面對面場景中，確保角色資訊不被他人偷看
3. **投票流程的便捷性** — 阿瓦隆投票頻繁，操作必須簡單快速

### 不要做的事

1. 不要試圖做線上對戰版本 — 這違反產品定位且進入紅海
2. 不要在 V1 加入太多可選角色 — 先做好基礎配置，根據反饋再擴展
3. 不要在規則上做創新 — 忠實還原官方規則，玩家期望的是標準阿瓦隆
