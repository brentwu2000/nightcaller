## 概述

### 遊戲簡介

獵巫鎮 Salem 1692 是一款社交推理桌遊，屬於「隱藏身分 / 陣營對抗」類型。玩家人數 4-12 人，每局約 20-40 分鐘。背景設定在 1692 年薩勒姆鎮，玩家分為村民與巫師兩個陣營，透過指控、審判、夜晚殺戮與共謀等機制進行對抗。

本 App 以 **H5 PWA** 形式提供「語音主持人」功能，**一台設備放在桌上**即可自動主持整場遊戲。玩家使用**實體卡牌**進行遊戲操作，App 負責**語音播報、階段流轉、計時與音效**。

### 產品定位

**單裝置離線語音主持工具** — 掃碼即用，無需安裝，離線可用。

### 使用模式

```
選擇遊戲 → 設定人數 → 發牌（實體卡牌）→ 開始
→ 設備放桌上，語音自動播報
→ 玩家用實體卡牌進行指控、出牌、投票
→ App 負責：語音主持 + 階段流轉 + 計時 + 音效
```

### 陣營與角色定義

#### 陣營

| 陣營 | 勝利條件 |
|------|----------|
| 村民 (Villagers) | 揭開所有「巫師」身分卡 |
| 巫師 (Witches) | 殺死所有村民，或將所有人同化為巫師 |

#### 角色（可選進階模組）

| 角色 | 能力 | 策略 |
|------|------|------|
| Abigail Williams (阿比蓋爾) | 棄牌換取指控清除 | 扮演最純潔的村民以獲取信任 |
| Tituba (提圖芭) | 黑色牌觸發時偷牌 | 利用混亂累積資源 |
| Giles Corey (吉爾斯) | 被殺後有一次豁免機會 | 引誘巫師浪費刀 |
| Mary Warren (瑪麗) | 無上限裝備藍色卡 | 把自己武裝成防禦堡壘 |

**關鍵概念**：身分是**動態**的。透過「共謀」事件，村民可能隨時墮落成為巫師。

### 卡牌系統

| 卡牌類型 | 顏色 | 功能 |
|----------|------|------|
| 指控卡 (Accusation) | 紅色 | 累積 7 點指控觸發強制審判 |
| 咒語卡 (Spell) | 藍色 | 防禦/連結效果（如 Piety 抵銷指控、Matchmaker 連結命運） |
| 行動卡 (Action) | 綠色 | 攻擊/干擾效果（如 Robbery 偷牌、Alibi 移除指控） |
| 事件卡 (Event) | 黑色 | 抽到時立即結算，不可控 |
| 試煉卡 (Tryal) | — | 包含「村民」或「巫師」身分的隱藏卡 |

### 核心價值主張

- **零主持門檻**：一台手機/平板放桌上，TTS 語音自動主持
- **掃碼即用**：H5 PWA 無需安裝，掃 QR Code 即可開始
- **離線可用**：PWA 離線快取，露營、密室等無網路環境都能使用
- **精準計時**：白天行動、夜晚階段計時精確，排除人工主持誤差
- **沉浸體驗**：語音台詞 + 音效 + 背景音樂三位一體，懸疑驚悚氛圍
- **動態身分**：共謀機制讓遊戲充滿變數，每局體驗不同

## 遊戲階段定義

```
[選擇遊戲模板] → [設定人數/參數] → [規則說明語音] →
┌──────────────────────────────────────────┐
│  遊戲循環（重複直到勝負分出）：            │
│  [白天行動] → [審判（若觸發）] →           │
│  [共謀事件] → [夜晚階段] → [黎明結算]     │
└──────────────────────────────────────────┘
→ [勝負結算語音] → (再來一局 or 結束)
```

### 階段一：遊戲設定

**目標**：選擇遊戲、設定人數與遊戲參數。

- 組織者選擇「獵巫鎮 Salem 1692」遊戲模板
- 設定玩家人數（4-12 人）
- 選擇是否啟用角色能力模組（進階規則）
- 調整語音設定（語速、音量）
- 調整白天行動計時（每人 30 秒 - 2 分鐘，預設 1 分鐘）
- App 顯示物品清單（試煉卡、指控卡、咒語卡、行動卡、事件卡、殺戮標記、星座卡）
- 可播放語音規則說明（可跳過）

### 階段二：白天行動 (Daytime)

**目標**：玩家輪流行動，語音引導每位玩家執行回合。

- 語音播報「白天到來，請開始行動」
- 依序提示每位玩家行動（二選一）：
  - **祈禱（抽牌）**：抽 2 張牌，若抽到黑色事件卡須立即結算
  - **指控與行動（出牌）**：打出紅色指控卡、藍色咒語卡或綠色行動卡
- 每位玩家有計時限制
- 若某位玩家累積 7 點指控，語音宣告「審判開始！」
- 組織者可暫停/恢復語音主持

### 階段三：審判 (Trial)

**目標**：當玩家累積 7 點指控時觸發強制審判。

- 語音播報「[玩家名] 被指控為巫師，審判即將開始」
- 語音引導被告辯護（計時）
- 語音引導投票「三、二、一，投票！」
- 組織者輸入投票結果（有罪/無罪）
- 若有罪：語音宣告翻開被告的所有試煉卡
- 若無罪：語音宣告無罪釋放，移除所有指控

### 階段四：共謀事件 (Conspiracy)

**目標**：語音引導隱藏的試煉卡傳遞，可能改變玩家陣營。

- 語音播報「共謀時刻到來。請所有人閉上眼睛」
- 語音指示「請每位玩家將一張隱藏的試煉卡傳給左手邊的玩家」
- 等待 10 秒讓玩家完成傳遞
- 語音播報「請查看你的試煉卡。若你收到巫師卡，你現在是巫師了」
- **核心規則**：一旦拿到「巫師」卡，該玩家立刻加入巫師陣營，不可告知他人

### 階段五：夜晚階段 (Night Phase)

**目標**：語音主持夜晚事件——巫師殺戮與警長守護。

#### A. 巫師殺戮

- 語音播報「夜幕降臨，請所有人閉上眼睛」
- 語音播報「巫師們，請睜開眼睛」
- 等待 10 秒讓巫師一致決定受害者
- 語音播報「巫師們，請閉上眼睛」

#### B. 警長守護（若有星座卡）

- 語音播報「擁有星座卡的守護者，請睜開眼睛」
- 等待 8 秒讓守護者指定保護對象
- 語音播報「守護者，請閉上眼睛」

### 階段六：黎明結算

**目標**：結算夜晚事件，判斷是否有玩家死亡。

- 語音播報「黎明到來，請所有人睜開眼睛」
- 組織者在 App 中輸入：受害者是否被守護
- 若未被守護：語音宣告「[玩家名] 在夜晚中被殺害。請翻開他的所有試煉卡」
- 若被守護：語音宣告「昨晚一切平安，守護者成功保護了目標」
- 系統檢查勝負條件

### 階段七：勝負結算

**目標**：語音播報勝負結果。

- 村民勝利條件：所有巫師身分卡被揭開
- 巫師勝利條件：所有村民被殺，或所有存活玩家都是巫師
- 語音播報勝利方結算台詞
- 顯示「再來一局」和「結束遊戲」按鈕
- 遊戲記錄自動儲存至本地（日期、人數、回合數、勝利陣營）

## User Stories

### US-01：選擇獵巫鎮遊戲模板

**As a** 桌遊組織者
**I want** 選擇「獵巫鎮 Salem 1692」遊戲模板，讓 App 幫我主持
**So that** 我不需要額外準備主持人，所有人都能參與遊戲

**Acceptance Criteria:**
- [ ] Given 使用者在首頁，When 點擊「獵巫鎮」遊戲模板，Then 進入遊戲設定頁
- [ ] Given 首頁顯示遊戲列表，Then 獵巫鎮有清楚的遊戲簡介與封面圖（1692 薩勒姆風格）
- [ ] Given 使用者首次使用，Then 顯示簡要的使用引導（一台設備放桌上即可）

**Edge Cases:**
- EC-1：PWA 尚未完成離線快取時，顯示載入進度
- EC-2：瀏覽器不支援 TTS 時，顯示相容性提示

**Priority:** P0
**Effort:** S

---

### US-02：設定玩家人數與遊戲參數

**As a** 組織者
**I want** 設定玩家人數（4-12 人）與遊戲參數
**So that** 語音主持的內容正確反映當局的遊戲設定

**Acceptance Criteria:**
- [ ] Given 進入遊戲設定頁，Then 顯示玩家人數選擇器（4-12 人）
- [ ] Given 選擇人數後，Then App 顯示對應的試煉卡配置（巫師/村民數量）
- [ ] Given 選擇人數後，Then 可輸入每位玩家暱稱（可選，用於語音點名）
- [ ] Given 4-6 人局，Then 配置 1 名巫師
- [ ] Given 7-9 人局，Then 配置 2 名巫師
- [ ] Given 10-12 人局，Then 配置 3 名巫師
- [ ] Given 設定完成，Then 可選擇是否啟用角色能力模組
- [ ] Given 設定完成，Then 可調整白天行動計時、辯護時間

**Edge Cases:**
- EC-1：人數低於 4 人時，開始按鈕 disabled 並顯示「最少需要 4 位玩家」
- EC-2：人數超過 12 人時，選擇器上限為 12

**Priority:** P0
**Effort:** S

### US-03：聽到清晰的語音主持

**As a** 玩家
**I want** 聽到清晰的語音指令引導遊戲各階段
**So that** 我能專注遊戲而不需要額外的人工主持

**Acceptance Criteria:**
- [ ] Given 夜晚階段開始，Then 語音清晰播報巫師睜眼/閉眼指令
- [ ] Given 白天階段開始，Then 語音依序提示每位玩家行動
- [ ] Given 審判觸發，Then 語音引導辯護與投票流程
- [ ] Given 共謀事件開始，Then 語音指示試煉卡傳遞
- [ ] Given 語音播報中，Then 語調帶有懸疑氛圍，夜晚/白天風格區分明顯
- [ ] Given 設備放桌上，Then 語音足夠清楚，2 公尺範圍內可聽清

**Edge Cases:**
- EC-1：TTS 引擎載入失敗時，顯示文字提示讓組織者手動朗讀
- EC-2：某些瀏覽器 TTS 聲音品質較差時，提供視覺主持模式備援

**Priority:** P0
**Effort:** M

### US-04：白天行動回合管理

**As a** 組織者
**I want** App 自動管理白天行動回合順序和計時
**So that** 遊戲節奏流暢，每位玩家都有公平的行動時間

**Acceptance Criteria:**
- [ ] Given 白天開始，Then 語音依序點名玩家行動
- [ ] Given 玩家行動中，Then 倒計時顯示（預設每人 1 分鐘）
- [ ] Given 玩家行動完成，When 組織者點擊「下一位」，Then 切換到下一位玩家
- [ ] Given 倒數歸零，Then 語音提示「時間到，輪到下一位」
- [ ] Given 玩家打出指控卡，When 累積達 7 點，Then 語音宣告審判觸發

**Edge Cases:**
- EC-1：玩家被淘汰後，自動跳過其回合
- EC-2：只剩極少存活玩家時，自動縮短行動時間

**Priority:** P0
**Effort:** M

### US-05：審判流程語音主持

**As a** 玩家
**I want** 審判時有清晰的語音引導辯護和投票
**So that** 審判過程公正有序

**Acceptance Criteria:**
- [ ] Given 指控累積觸發審判，Then 語音宣告被告身分
- [ ] Given 審判開始，Then 語音引導被告進行辯護（計時 1-3 分鐘）
- [ ] Given 辯護結束，Then 語音引導「三、二、一，投票！」
- [ ] Given 投票結束，When 組織者輸入結果，Then 語音播報有罪/無罪結果
- [ ] Given 有罪判決，Then 語音提示翻開被告試煉卡，並判斷勝負

**Edge Cases:**
- EC-1：被告使用 Alibi 卡移除指控，審判取消
- EC-2：同一回合多人同時達到 7 點指控，依順序逐一審判

**Priority:** P0
**Effort:** M

### US-06：共謀事件語音引導

**As a** 玩家
**I want** 共謀事件有清晰的語音引導
**So that** 試煉卡傳遞過程安全，不會洩漏身分

**Acceptance Criteria:**
- [ ] Given 共謀階段開始，Then 語音指示所有人閉眼
- [ ] Given 所有人閉眼，Then 語音指示傳遞試煉卡
- [ ] Given 傳遞完成，Then 語音提示查看新卡片
- [ ] Given 語音提示查看卡片，Then 播放提示音讓玩家知道可以偷偷查看
- [ ] Given 查看完成後，Then 語音提示「請收好你的試煉卡，準備進入夜晚」

**Edge Cases:**
- EC-1：某位玩家只剩一張試煉卡且為巫師卡，傳出後變回村民
- EC-2：新手玩家不理解傳遞方向，語音重複提示「傳給你左手邊的玩家」

**Priority:** P0
**Effort:** S

### US-07：夜晚階段語音主持

**As a** 玩家
**I want** 夜晚階段有完整的語音引導巫師殺戮和警長守護
**So that** 所有夜晚行動安全進行，不會洩漏身分

**Acceptance Criteria:**
- [ ] Given 夜晚開始，Then 語音播報「請所有人閉上眼睛，夜幕降臨」
- [ ] Given 巫師階段，Then 語音播報「巫師們，請睜開眼睛」，等待 10 秒
- [ ] Given 巫師行動結束，Then 語音播報「巫師們，請閉上眼睛」
- [ ] Given 有星座卡存在，Then 語音播報守護者睜眼指令，等待 8 秒
- [ ] Given 所有夜晚行動完成，Then 語音播報黎明結算

**Edge Cases:**
- EC-1：無巫師存活（全部被審判揭露），夜晚自動跳過殺戮
- EC-2：巫師與守護者為同一人（擁有巫師卡又有星座卡），兩次都需睜眼

**Priority:** P0
**Effort:** M

### US-08：暫停/恢復語音主持

**As a** 組織者
**I want** 暫停/恢復語音主持
**So that** 有人遲疑或需要暫離時，遊戲可以暫時中斷

**Acceptance Criteria:**
- [ ] Given 任何遊戲階段進行中，When 點擊暫停按鈕，Then 語音暫停、計時暫停，螢幕顯示暫停狀態
- [ ] Given 暫停狀態，When 點擊恢復按鈕，Then 從暫停處繼續語音和計時
- [ ] Given 暫停按鈕，Then 按鈕足夠大（最小 64x64px），方便緊急操作
- [ ] Given 暫停中，Then 背景音樂同步暫停

**Edge Cases:**
- EC-1：暫停超過 10 分鐘，App 顯示「是否繼續遊戲？」確認
- EC-2：螢幕鎖定後重新開啟，App 恢復暫停狀態

**Priority:** P0
**Effort:** S

### US-09：背景音樂和音效

**As a** 玩家
**I want** 有背景音樂和音效增加懸疑氛圍
**So that** 遊戲體驗更加沉浸和有趣

**Acceptance Criteria:**
- [ ] Given 夜晚階段開始，Then 播放夜晚氛圍背景音樂（陰風、烏鴉叫聲、教堂鐘聲）
- [ ] Given 白天階段開始，Then 切換為白天氛圍背景音樂（市集喧嘩、火堆噼啪）
- [ ] Given 審判觸發，Then 播放審判鼓聲音效
- [ ] Given 共謀階段，Then 播放詭異低沉的氛圍音樂
- [ ] Given 有玩家死亡，Then 播放死亡音效（教堂鐘聲）
- [ ] Given TTS 語音播報中，Then 背景音樂自動降至 10% 音量

**Edge Cases:**
- EC-1：瀏覽器禁止自動播放音訊時，顯示「點擊啟用音效」按鈕
- EC-2：音訊檔案載入失敗時，遊戲仍可正常進行（語音主持為核心）

**Priority:** P1
**Effort:** M

### US-10：調整語速和音量

**As a** 組織者
**I want** 調整語速和音量

**Acceptance Criteria:**
- [ ] Given 遊戲設定頁或進行中，Then 可調整語速（0.8x - 1.5x，預設 1.0x）

- [ ] Given 遊戲設定頁或進行中，Then 可調整背景音樂音量（0% - 100%，預設 30%）
- [ ] Given 調整設定後，Then 設定儲存至 localStorage，下次開啟自動載入

**Edge Cases:**
- EC-1：語速過快導致語音不清晰時，提示建議範圍
- EC-2：音量為 0 時，顯示「靜音模式」提示

**Priority:** P1
**Effort:** S

### US-11：無網路環境使用

**As a** 組織者
**I want** 在無網路環境下也能使用
**So that** 不受網路限制，任何地方都能主持遊戲

**Acceptance Criteria:**
- [ ] Given PWA 已安裝/首次開啟並快取完成，Then 離線狀態下可完整進行遊戲
- [ ] Given 離線狀態，Then TTS 語音正常播報
- [ ] Given 離線狀態，Then 音效和背景音樂正常播放
- [ ] Given 離線狀態，Then 遊戲記錄正常儲存至本地

**Edge Cases:**
- EC-1：瀏覽器不支援 Service Worker 時，顯示提示
- EC-2：localStorage 空間不足時，提示清理舊記錄

**Priority:** P0
**Effort:** M

---

### US-12：查看遊戲歷史記錄

**As a** 組織者
**I want** 查看遊戲歷史記錄
**So that** 可以追蹤玩了多少局、各陣營勝率等統計

**Acceptance Criteria:**
- [ ] Given 結算完成，Then 本局記錄自動儲存（日期、人數、回合數、勝利陣營、局時長）
- [ ] Given 進入歷史記錄頁，Then 顯示對局列表，時間倒序排列
- [ ] Given 歷史記錄頁，Then 顯示統計概覽：總局數、村民勝率、巫師勝率
- [ ] Given 使用者清除瀏覽器資料，Then 歷史記錄消失，顯示空狀態引導

**Edge Cases:**
- EC-1：記錄超過 200 筆時，自動清理最早的記錄
- EC-2：不同瀏覽器之間記錄不互通

**Priority:** P2
**Effort:** S

## Edge Cases 清單

### EC-01：全員被同化為巫師

**情境**：透過多輪共謀事件，所有存活玩家都收到了巫師卡

**影響**：巫師陣營自動獲勝

**預期行為**：
- 共謀結算後系統檢查勝負條件
- 語音播報「黑暗籠罩了整個薩勒姆鎮，所有人都已墮落。巫師陣營獲勝！」
- 自動進入結算階段

---

### EC-02：無辜村民被審判處刑

**情境**：投票判定有罪的玩家實際上是村民

**影響**：翻開試煉卡顯示為村民，村民方損失一名成員

**預期行為**：
- 語音播報「翻開試煉卡... [玩家名] 是清白的村民！薩勒姆鎮失去了一名無辜的靈魂」
- 該玩家出局，遊戲繼續
- 系統檢查勝負條件

---

### EC-03：巫師全部被揭露但仍有巫師卡在流通

**情境**：所有已知巫師被處刑，但透過共謀可能產生新巫師

**影響**：遊戲仍需繼續直到確認所有巫師卡被揭開

**預期行為**：
- 組織者需在 App 中標記所有巫師卡的狀態
- 若所有巫師卡均已翻開，觸發村民勝利
- 語音提供提示「請確認所有巫師卡是否已揭露」

---

### EC-04：裝置靜音或音量過低

**情境**：語音主持進行中，設備處於靜音模式或音量過低

**預期行為**：
- App 啟動前偵測音量狀態，若偵測到靜音/低音量，顯示提示
- 建議使用者調高設備音量
- 靜音模式下可改用螢幕顯示大字提示（視覺主持模式）

---

### EC-05：玩家中途離桌

**情境**：遊戲進行中，有一位玩家需要暫離

**預期行為**：
- 組織者按暫停按鈕，等待玩家返回
- 暫停中語音提示「遊戲暫停中，請稍候」
- 恢復後從暫停處繼續
- 若玩家離開不回，組織者可標記該玩家退出

---

### EC-06：夜晚巫師無法達成共識

**情境**：巫師睜眼後無法一致決定殺害目標

**預期行為**：
- 夜晚行動有 10 秒計時限制
- 若超時，語音播報「時間到，巫師們請閉眼」
- 組織者可選擇「無人被殺」或「重新夜晚巫師環節」

---

### EC-07：設備電量不足

**情境**：語音主持進行中，手機電量耗盡

**預期行為**：
- App 開始遊戲前顯示電量警告（低於 20%）
- PWA 重新開啟後需重新開始新局（單局不保存中間狀態）

## 技術說明

### 平台選型

| 項目 | 選型 | 理由 |
|------|------|------|
| 框架 | Vue 3 + Vite | 輕量快速，適合 PWA |
| 語音 | Web Speech API (TTS) | 瀏覽器內建，離線可用 |
| 音效 | Howler.js | 瀏覽器音訊處理 |
| 儲存 | IndexedDB (Dexie.js) / localStorage | 本地離線儲存 |
| 部署 | PWA + Service Worker | 離線快取、掃碼即用 |
| 狀態管理 | Pinia | Vue 3 官方推薦 |

### 與奶酪大盜的差異

| 比較項 | 奶酪大盜 | 獵巫鎮 |
|--------|----------|--------|
| 遊戲時長 | ~10 分鐘/局 | ~20-40 分鐘/局 |
| 人數 | 4-8 人 | 4-12 人 |
| 夜晚複雜度 | 單一叫號 (1-6) | 巫師殺戮 + 警長守護 |
| 陣營變化 | 無 | 動態（共謀改變陣營） |
| 回合制 | 單回合 | 多回合循環 |
| 審判機制 | 一次投票 | 指控累積觸發審判 |
| 主持複雜度 | 低（線性流程） | 高（多分支、條件判斷） |

### 不需要的後端服務

- 不需要帳號系統
- 不需要資料庫
- 不需要 API 伺服器
- 不需要即時通訊
- 所有資料儲存在本地瀏覽器
