## 概述

### 遊戲簡介

奶酪大盜（Cheese Thief）是一款社交推理桌遊，屬於「隱藏角色 / 社交欺騙」類型。玩家人數 4-8 人，每局約 10 分鐘。遊戲透過骰子決定「起床時間」作為核心推理線索，搭配隱藏角色機制，讓所有玩家在不淘汰任何人的前提下進行推理與投票。

本 App 以 **H5 PWA** 形式提供「語音主持人」功能，**一台設備放在桌上**即可自動主持整場遊戲。玩家使用**實體卡牌和骰子**進行角色分配與遊戲操作，App 僅負責**語音播報、計時與音效**。

### 產品定位

**單裝置離線語音主持工具** — 掃碼即用，無需安裝，離線可用。

### 使用模式

```
舊模式（已廢棄）：每人一台手機
  建立房間 → 分享代碼 → 每人加入 → 各自看角色 → 同步遊戲

新模式（v2.0）：一台設備當主持人
  選擇遊戲 → 設定人數 → 開始
  → 設備放桌上，語音自動播報
  → 玩家用實體骰子和角色卡
  → App 只負責：語音主持 + 計時 + 音效
```

### 角色定義

| 角色 | 人數 | 陣營 | 能力 |
|------|------|------|------|
| 奶酪大盜 | 1 人 | 大盜陣營 | 偷走奶酪標記；可指定共犯（5-8 人局） |
| 共犯 | 0-2 人 | 大盜陣營 | 被大盜觸碰後成為共犯；5-8 人局才存在 |
| 貪睡鼠 | 其餘玩家 | 老鼠陣營 | 獨自醒來時可偷看他人骰子；需找出大盜 |

### 核心價值主張

- **零主持門檻**：一台手機/平板放桌上，TTS 語音自動主持
- **掃碼即用**：H5 PWA 無需安裝，掃 QR Code 即可開始
- **離線可用**：PWA 離線快取，露營、密室等無網路環境都能使用
- **精準計時**：夜晚階段每號碼間隔 5 秒，排除人工主持誤差
- **沉浸體驗**：語音台詞 + 音效 + 背景音樂三位一體
- **快速重玩**：一局 10 分鐘，結算後立即再來一局

### 不包含的功能

以下功能在 v2.0 中**已移除**，因為新模式使用實體道具進行：

- ~~線上建房 / 加入房間~~ — 不需要，單裝置使用
- ~~數位角色分配~~ — 用實體角色卡
- ~~數位投票系統~~ — 面對面舉手/指人
- ~~數位骰子~~ — 用實體骰子和骰盅
- ~~共犯指定功能~~ — 用實體動作（觸碰肩膀）
- ~~偷看骰子功能~~ — 用實體骰子偷看
- ~~帳號系統~~ — 不需要
- ~~房間代碼~~ — 不需要
- ~~線上匹配~~ — 不需要
- ~~後端 API~~ — 純前端離線應用

## 遊戲階段定義

```
[選擇遊戲模板] → [設定人數/參數] → [規則說明語音] →
[夜晚語音主持 (1→6叫號)] → [白天討論計時] →
[投票計時] → [結算語音] → (再來一局 or 結束)
```

### 階段一：遊戲設定

**目標**：選擇遊戲、設定人數與遊戲參數。

- 組織者選擇「奶酪大盜」遊戲模板
- 設定玩家人數（4-8 人）
- 調整語音設定（語速、音量）
- 調整討論時間（1-10 分鐘，預設 3 分鐘）
- 調整投票時間（30 秒 - 2 分鐘，預設 1 分鐘）
- App 顯示物品清單（骰子、骰盅、角色卡、奶酪標記）
- 可播放語音規則說明（可跳過）

### 階段二：夜晚語音主持

**目標**：語音主持人依序喊 1 到 6，引導玩家行動。

- 語音播報「請所有人閉上眼睛，夜晚降臨」
- 依序播放「骰子數字是 1 的玩家，請睜開眼睛」到 6 號
- 每個號碼播報後等待 5 秒
- 所有號碼播報完畢後，語音宣告「天亮了，請所有人睜開眼睛」
- 設備放桌上面朝下或螢幕關閉，純靠語音引導
- 組織者可暫停/恢復語音主持

### 階段三：白天討論計時

**目標**：計時討論階段，語音提示時間進度。

- App 開始倒數計時（預設 3 分鐘）
- 最後 30 秒語音提醒「即將進入投票」
- 倒數歸零時語音播報「討論時間結束，請準備投票」
- 組織者可提前結束討論或暫停計時

### 階段四：投票計時

**目標**：計時投票階段（面對面舉手/指人）。

- 語音播報「三、二、一，指！」引導同時投票
- 投票計時倒數
- 玩家面對面指人，App 不參與投票記錄

### 階段五：結算語音

**目標**：語音播報結算提示，由組織者口頭確認結果。

- 語音詢問「大盜是否被指認？」
- 組織者根據實際投票結果在 App 上選擇勝負
- 語音播報勝利方結算台詞
- 顯示「再來一局」和「結束遊戲」按鈕
- 遊戲記錄自動儲存至本地（日期、人數、勝利陣營）

## User Stories

### US-01：選擇遊戲模板

**As a** 桌遊組織者
**I want** 選擇「奶酪大盜」遊戲模板，讓 App 幫我主持
**So that** 我不需要額外準備主持人，所有人都能參與遊戲

**Acceptance Criteria:**
- [ ] Given 使用者在首頁，When 點擊「奶酪大盜」遊戲模板，Then 進入遊戲設定頁
- [ ] Given 首頁顯示遊戲列表，Then 奶酪大盜有清楚的遊戲簡介與封面圖
- [ ] Given 使用者首次使用，Then 顯示簡要的使用引導（一台設備放桌上即可）

**Edge Cases:**
- EC-1：PWA 尚未完成離線快取時，顯示載入進度
- EC-2：瀏覽器不支援 TTS 時，顯示相容性提示

**Priority:** P0
**Effort:** S

---

### US-02：聽到清晰語音指令

**As a** 玩家
**I want** 聽到清晰的語音指令「X 號請起床」
**So that** 我能在閉眼狀態下知道何時該行動

**Acceptance Criteria:**
- [ ] Given 夜晚階段開始，When App 播報，Then 語音清晰、音量充足，在安靜環境中 2 公尺範圍內可聽清
- [ ] Given 語音播報，Then 每個號碼之間間隔 5 秒，節奏穩定
- [ ] Given 夜晚階段語音，Then 音調低沉帶有神秘感，與白天語音風格有明確區別
- [ ] Given 設備放桌上，Then 語音足夠清楚，不需要玩家拿著手機

**Edge Cases:**
- EC-1：TTS 引擎載入失敗時，顯示文字提示讓組織者手動朗讀
- EC-2：某些瀏覽器 TTS 聲音品質較差時，提供預錄音檔備援方案

**Priority:** P0
**Effort:** M

---

### US-03：設定玩家人數

**As a** 組織者
**I want** 設定玩家人數（4-8 人）來調整語音腳本
**So that** 語音主持的內容正確反映當局的遊戲設定

**Acceptance Criteria:**
- [ ] Given 進入遊戲設定頁，Then 顯示玩家人數選擇器（4-8 人）
- [ ] Given 選擇人數後，Then App 顯示對應的角色卡組成（大盜/共犯/貪睡鼠）和物品清單
- [ ] Given 4 人局，Then 語音腳本不包含共犯相關說明
- [ ] Given 5 人以上，Then 語音腳本包含共犯指定說明
- [ ] Given 設定完成，Then 可調整討論時間（1-10 分鐘）和投票時間

**Edge Cases:**
- EC-1：人數低於 4 人時，開始按鈕 disabled 並顯示「最少需要 4 位玩家」
- EC-2：人數超過 8 人時，選擇器上限為 8

**Priority:** P0
**Effort:** S

---

### US-04：背景音樂和音效

**As a** 玩家
**I want** 有背景音樂和音效增加氛圍
**So that** 遊戲體驗更加沉浸和有趣

**Acceptance Criteria:**
- [ ] Given 夜晚階段開始，Then 播放夜晚氛圍背景音樂（蟲鳴、微風）
- [ ] Given 白天階段開始，Then 切換為白天氛圍背景音樂（鳥鳴、陽光感）
- [ ] Given 倒計時最後 10 秒，Then 播放急促的計時音效
- [ ] Given 結算階段，Then 播放對應的勝利/失敗音效
- [ ] Given TTS 語音播報中，Then 背景音樂自動降至 10% 音量

**Edge Cases:**
- EC-1：瀏覽器禁止自動播放音訊時，顯示「點擊啟用音效」按鈕
- EC-2：音訊檔案載入失敗時，遊戲仍可正常進行（語音主持為核心）

**Priority:** P1
**Effort:** M

### US-05：暫停/恢復語音主持

**As a** 組織者
**I want** 暫停/恢復語音主持
**So that** 有人遲疑或需要暫離時，遊戲可以暫時中斷

**Acceptance Criteria:**
- [ ] Given 任何遊戲階段進行中，When 點擊暫停按鈕，Then 語音暫停、計時暫停，螢幕顯示暫停狀態
- [ ] Given 暫停狀態，When 點擊恢復按鈕，Then 從暫停處繼續語音和計時
- [ ] Given 暫停按鈕，Then 按鈕足夠大（最小 64x64px），方便在緊急情況下快速操作
- [ ] Given 暫停中，Then 背景音樂同步暫停

**Edge Cases:**
- EC-1：暫停超過 10 分鐘，App 顯示「是否繼續遊戲？」確認
- EC-2：螢幕鎖定後重新開啟，App 恢復暫停狀態

**Priority:** P0
**Effort:** S

## Edge Cases 清單

### EC-01：骰子數字完全相同（全員同號）

**情境**：所有 4-8 位玩家恰好擲出相同的骰子數字

**影響**：夜晚階段所有人同時醒來，貪睡鼠不能偷看（因為有人同時醒），大盜醒來偷走奶酪

**預期行為**：
- 遊戲正常進行，僅有一個號碼有多人同時醒來
- 語音主持照常播報所有 1-6 號碼
- App 不知道玩家的骰子分布，照標準流程主持

---

### EC-02：奶酪大盜忘記偷奶酪

**情境**：夜晚階段大盜被叫醒，但因緊張或分心未拿走奶酪標記

**影響**：白天討論結束後奶酪仍在原位

**預期行為**：
- 結算階段語音提示組織者確認奶酪狀態
- 組織者自行判斷如何處理（重玩/無效局/忽略）
- App 提供「重新夜晚」和「直接結算」兩個選項

---

### EC-03：裝置靜音或音量過低

**情境**：語音主持進行中，設備處於靜音模式或音量過低

**預期行為**：
- App 啟動前偵測音量狀態，若偵測到靜音/低音量，顯示提示
- 建議使用者調高設備音量
- 靜音模式下可改用螢幕顯示大字號碼（視覺主持模式）

---

### EC-04：玩家中途離桌

**情境**：遊戲進行中，有一位玩家需要暫離

**預期行為**：
- 組織者按暫停按鈕，等待玩家返回
- 暫停中語音提示「遊戲暫停中，請稍候」
- 恢復後從暫停處繼續

---

### EC-05：設備電量不足

**情境**：語音主持進行中，手機電量耗盡

**預期行為**：
- App 開始遊戲前顯示電量警告（低於 20%）
- PWA 重新開啟後可以直接開始新一局（無狀態恢復需求，因為單局時間短）

---

### EC-06：瀏覽器 TTS 不支援中文

**情境**：某些瀏覽器或設備不支援中文 TTS

**預期行為**：
- App 啟動時檢測 TTS 可用性
- 若不支援中文，切換為預錄音檔模式
- 若預錄音檔也不可用，切換為視覺主持模式（螢幕顯示文字）

## 技術說明

### 平台選型

| 項目 | 選型 | 理由 |
|------|------|------|
| 框架 | Vue 3 + Vite | 輕量快速，適合 PWA |
| 語音 | Web Speech API (TTS) | 瀏覽器內建，離線可用 |
| 音效 | Web Audio API / Howler.js | 瀏覽器音訊處理 |
| 儲存 | localStorage / IndexedDB | 本地離線儲存 |
| 部署 | PWA + Service Worker | 離線快取、掃碼即用 |
| 狀態管理 | Pinia | Vue 3 官方推薦 |

### 不需要的後端服務

- 不需要帳號系統
- 不需要資料庫
- 不需要 API 伺服器
- 不需要即時通訊
- 所有資料儲存在本地瀏覽器
