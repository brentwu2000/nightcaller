## 1. 目標用戶分析

### 1.1 阿瓦隆典型玩家群體

| 分群 | 描述 | 佔比 | 核心需求 |
|------|------|------|----------|
| 社交推理核心玩家 | 熟悉阿瓦隆規則，每週固定局，追求策略深度 | 35% | 快速開局、角色配置彈性、流程不出錯 |
| 桌遊社團組織者 | 大學桌遊社、公司團建負責人，需帶新手入門 | 30% | 語音引導新手、規則說明自動化、降低主持門檻 |
| 輕度社交玩家 | 聚會時偶爾玩，不完全熟悉規則 | 25% | 不用背規則、跟著語音提示走即可 |
| 桌遊店桌主 | 桌遊店內帶局的工作人員 | 10% | 專業穩定、支援進階角色、減少人為失誤 |

### 1.2 與奶酪大盜的用戶重疊度

```
                奶酪大盜              阿瓦隆
              (輕鬆派對)          (社交推理)
            ┌───────────┐     ┌───────────┐
            │           │     │           │
            │  休閒玩家  │     │ 推理核心  │
            │   (40%)   ├─────┤  玩家     │
            │           │重疊 │  (35%)    │
            │           │20%  │           │
            └───────────┘     └───────────┘
                  │                 │
                  └────────┬────────┘
                           │
                     共同特徵：
                  - 線下面對面聚會
                  - 需要語音主持
                  - 一台設備搞定
```

**重疊分析**:

- **用戶重疊度約 20%**：奶酪大盜偏向輕鬆家庭/派對遊戲，阿瓦隆偏向策略推理，兩者吸引的核心玩家群體不同
- **場景重疊度約 60%**：都是線下聚會場景，同一群人可能先玩奶酪大盜暖場，再轉阿瓦隆
- **價值重疊度 100%**：「一台設備當主持人」的核心價值主張完全一致

### 1.3 新增用戶獲取機會

| 機會 | 說明 | 預估新增用戶 |
|------|------|-------------|
| 社交推理玩家群體 | 阿瓦隆/狼人殺玩家基數遠大於奶酪大盜 | +300% |
| 大學桌遊社 | 阿瓦隆是桌遊社最常見的遊戲之一 | 每社 20-50 人 |
| 桌遊店場景 | 阿瓦隆是桌遊店點桌率最高的遊戲之一 | 每店 100+ 潛在用戶 |
| 企業團建 | 阿瓦隆常被用於團隊建設活動 | 中低頻但口碑傳播強 |

**關鍵洞察**: 阿瓦隆的知名度和玩家基數遠超奶酪大盜，上線阿瓦隆預計能帶來 3-5 倍的用戶增長，且這批用戶本身就是桌遊重度用戶，留存潛力更高。

## 2. 競品分析

### 2.1 現有競品對標

| 競品 | 平台 | 優勢 | 劣勢 | 定價 |
|------|------|------|------|------|

### 2.2 競品缺口分析

```
現有競品的共同問題：

1. 功能碎片化
   - 大多只做「開場語音」（閉眼睜眼階段）
   - 沒有完整的任務投票流程主持
   - 沒有刺殺梅林流程

2. 體驗粗糙
   - 無背景音效/氛圍音樂
   - TTS 語音生硬，缺乏遊戲感
   - UI 設計停留在 2015 年水準

3. 場景錯位
   - 線上版本要求每人一台設備（非面對面場景）
   - 桌面版本不適合桌遊聚會攜帶

4. 維護停滯
   - 多數 App 超過 2 年未更新
   - 不支援新設備/新系統
```

### 2.3 我們的差異化優勢

| 差異化維度 | 競品現狀 | 我們的定位 |
|-----------|---------|-----------|
| **流程完整度** | 僅開場語音 | 完整遊戲流程主持（身份確認 > 組隊 > 投票 > 任務 > 刺殺） |
| **語音體驗** | 單調 TTS / 錄音 | Web Speech API + 語音腳本設計，搭配音效營造氛圍 |
| **使用場景** | 線上/需多設備 | 一台設備放桌上，面對面場景最佳化 |
| **技術架構** | 需下載 App / 需網路 | PWA 離線可用，掃碼即開，免安裝 |
| **多遊戲整合** | 每款遊戲一個 App | 一個 App 涵蓋多款桌遊（已有奶酪大盜） |
| **角色配置** | 固定配置 | 彈性角色配置，支援自訂組合 |
| **UI/UX** | 過時簡陋 | 現代化行動端設計，暗色氛圍主題 |

## 3. 功能優先級排序 (MoSCoW)

### 3.1 Must Have (v1.0 -- 核心可玩)

這些功能缺少任何一項，遊戲就無法正常進行：

| 功能 | 說明 | 驗收標準 |
|------|------|----------|
| **玩家人數設定** | 支援 5-10 人 | 輸入人數後自動推薦角色配置 |
| **基礎角色分配** | 梅林、派西維爾、忠臣、莫甘娜、刺客、爪牙 | 隨機分配，傳閱確認身份 |
| **身份確認流程（夜晚階段）** | 語音引導閉眼/睜眼流程 | TTS 播報完整開場流程，各角色依序確認資訊 |
| **隊長輪替** | 顯示當前隊長，自動輪替 | 視覺化標示，語音播報隊長姓名 |
| **組隊階段** | 隊長選擇出任務的玩家 | 顯示任務需求人數，隊長操作選人 |
| **公投（贊成/反對）** | 全體投票是否同意此隊伍 | 計票並播報結果，連續 5 次否決則邪惡獲勝 |
| **任務執行** | 出任務的玩家秘密投票（成功/失敗） | 傳閱投票，播報任務結果（不透露誰投失敗） |
| **任務追蹤板** | 顯示 5 輪任務的成功/失敗狀態 | 視覺化任務軌道，第 4 輪（7人+）需 2 失敗 |
| **勝負判定** | 3 輪任務成功/失敗即結束 | 自動判定並播報結果 |
| **刺殺梅林** | 邪惡方在好人贏時可刺殺梅林 | 刺客選擇目標，揭曉結果 |
| **任務人數配置表** | 根據玩家人數自動設定每輪任務所需人數 | 完全遵照官方規則表 |

### 3.2 Should Have (v1.1 -- 體驗提升)

遊戲可以玩但有了這些會明顯更好：

| 功能 | 說明 | Impact |
|------|------|--------|
| **進階角色：奧伯倫** | 邪惡方但不知道其他壞人 | 增加遊戲變化度 |
| **進階角色：莫德雷德** | 不被梅林看到的邪惡 | 平衡好人方優勢 |
| **進階角色：好蘭斯洛特/壞蘭斯洛特** | 互相知道身份的對立角色 | 增加策略深度 |
| **背景音效/氛圍音樂** | 夜晚階段播放神秘音效 | 大幅提升沉浸感 |
| **語音風格選擇** | 嚴肅/歡樂/中世紀風格語音腳本 | 適應不同聚會氛圍 |
| **計時器** | 討論時間倒數 | 控制遊戲節奏 |
| **遊戲歷史記錄** | 記錄每場遊戲結果 | 回顧與統計 |

### 3.3 Could Have (v1.2 -- 進階功能)

錦上添花，有資源再做：

| 功能 | 說明 | Impact |
|------|------|--------|
| **湖中女士規則** | 第 2/3/4 輪結束後可查驗玩家陣營 | 官方變體規則 |
| **自訂角色配置** | 玩家自由選擇啟用哪些角色 | 高階玩家需求 |
| **統計分析** | 各角色勝率、各位置勝率 | 數據愛好者 |
| **遊戲回放** | 回放整場遊戲的關鍵決策點 | 覆盤學習 |
| **多語言支援** | 英文/簡中語音腳本 | 擴大用戶群 |

### 3.4 Won't Have (不做)

明確不在範圍內，避免功能蔓延：

| 功能 | 不做原因 |
|------|---------|
| 線上對戰/匹配 | 違反離線優先設計原則 |
| 多設備同步 | 一台設備放桌上是核心場景 |
| 用戶帳號/登入 | 無需後端，降低使用門檻 |
| AI 玩家/機器人 | 面對面社交遊戲，不需要 AI 對手 |
| 文字聊天 | 面對面場景不需要 |
| 排行榜/積分系統 | 非競技場景 |

## 4. 成功指標 (KPI)

### 4.1 北極星指標

**每週阿瓦隆遊戲完成場次 (Weekly Completed Avalon Games)**

> 定義：過去 7 天內完整走完勝負判定（含刺殺梅林階段）的阿瓦隆遊戲場次

> 為何選此指標：
> 1. 直接反映產品核心價值（成功主持一場阿瓦隆）
> 2. 「完成」而非「開始」，確保流程品質
> 3. 與留存高度相關 -- 完成一場好遊戲的人會再來

### 4.2 核心指標儀表板

| 指標類別 | 指標 | 目標 (v1.0) | 目標 (v1.1) | 說明 |
|---------|------|-----------|-----------|------|
| **品質** | 遊戲完成率 | > 75% | > 85% | 開始的遊戲成功完成的比例 |
| **品質** | 每場平均時長 | 20-40 分鐘 | 20-40 分鐘 | 過短表示中斷，過長表示卡關 |
| **黏性** | 重複遊玩率 | > 40% | > 55% | 玩過一場後 7 天內再玩的比例 |
| **規模** | 每場平均人數 | > 6 人 | > 7 人 | 反映社交裂變，阿瓦隆理想人數 7-8 |
| **滿意度** | NPS | > 45 | > 55 | 推薦意願 |
| **功能** | 進階角色使用率 | -- | > 30% | v1.1 上線後追蹤 |
| **跨遊戲** | 多遊戲用戶比例 | > 15% | > 25% | 同時玩過奶酪大盜和阿瓦隆 |

### 4.3 角色使用追蹤

追蹤每個角色的使用頻率，用於指導後續開發優先級：

```
追蹤項目：
- 各角色被選用的頻率
- 各角色配置組合的勝率分佈
- 最受歡迎的角色配置（幾人局 + 哪些角色）
- 遊戲放棄時的階段分佈（哪個階段最容易放棄）
```

### 4.4 技術健康指標

| 指標 | 目標 | 說明 |
|------|------|------|
| TTS 播報成功率 | > 98% | 語音合成是否正常運作 |
| 首次載入時間 | < 3 秒 | PWA 首次開啟速度 |
| 離線可用率 | 100% | Service Worker 快取覆蓋率 |
| 崩潰率 | < 0.5% | 遊戲流程中斷的比例 |

## 5. 推廣策略

### 5.1 利用阿瓦隆知名度

阿瓦隆本身具有強大的品牌知名度，不需要從零教育市場：

```
策略：把推廣重心放在「痛點共鳴」而非「遊戲介紹」

核心訊息：
"每次玩阿瓦隆，都在煩誰來當主持人？"
"開場白又唸錯了？讓語音主持人來搞定"
"不用再怕主持人偷看身份牌了"
```

### 5.2 推廣管道與策略

#### Phase 1: 種子用戶 (v1.0 上線後 0-4 週)

| 管道 | 具體做法 | 預期效果 |
|------|---------|---------|
| PTT 桌遊版 (BG板) | 發文分享工具，強調免費/免安裝 | 200-500 種子用戶 |
| Dcard 桌遊/遊戲看板 | 圖文心得分享，附 QR Code 連結 | 300-800 用戶 |
| Facebook 桌遊社團 | 台灣桌遊愛好者、各地桌遊聚會社團 | 500-1000 用戶 |
| 大學桌遊社 | 直接聯繫各校桌遊社，提供試用邀請 | 每社 20-50 人 |

#### Phase 2: 口碑擴散 (4-12 週)

| 管道 | 具體做法 | 預期效果 |
|------|---------|---------|
| 桌遊 YouTuber/實況主 | 邀請桌遊實況主使用工具進行直播 | 品牌曝光 + 信任背書 |
| 桌遊店合作 | 提供桌遊店免費使用，換取推薦 | 每店帶來 50-100 用戶 |
| Line 桌遊群組 | 在桌遊約局群組中自然推薦 | 穩定細流 |

#### Phase 3: 自然成長 (12 週+)

| 管道 | 具體做法 | 預期效果 |
|------|---------|---------|
| SEO | 「阿瓦隆線上主持」「阿瓦隆語音工具」關鍵字 | 長尾搜尋流量 |
| App 內跨遊戲推薦 | 奶酪大盜用戶看到阿瓦隆推薦 | 交叉導流 |
| 社交裂變 | 一人開局，同桌朋友自然認識產品 | 最強成長引擎 |

### 5.3 口碑傳播設計

阿瓦隆天然具有社交裂變屬性：一場遊戲 5-10 人，只需 1 人知道這個工具，同桌所有人都會體驗到。

```
裂變路徑：
1 個用戶開局 → 7 個同桌朋友體驗
→ 其中 2-3 人下次自己的局也使用
→ 每人再帶 7 個新朋友體驗
→ 指數級成長

設計要點：
- 遊戲結束畫面加入「分享給朋友」按鈕
- 提供可分享的遊戲結果卡片（不含劇透，只含統計）
- 首次使用引導極簡化（10 秒內開始第一場遊戲）
- PWA 加入主畫面提示（方便下次快速開啟）
```

### 5.4 內容行銷素材

| 素材類型 | 內容 | 目的 |
|---------|------|------|
| 短影片 | 「30 秒看懂語音主持人怎麼用」 | 降低理解門檻 |
| 對比圖 | 「人工主持 vs 語音主持」痛點對比 | 痛點共鳴 |
| 教學文 | 「阿瓦隆新手入門 + 主持工具推薦」 | SEO + 實用價值 |
| 角色配置推薦 | 「7人局最佳角色配置」 | 實用內容吸引目標用戶 |

## 6. 功能路線圖

### v1.0: 基礎阿瓦隆 (核心可玩)

**預計開發時間**: 3-4 週
**目標**: 能完整玩一場標準阿瓦隆

```
包含角色：
- 梅林 (Merlin)           -- 知道所有壞人（除莫德雷德）
- 派西維爾 (Percival)     -- 知道梅林（和莫甘娜）
- 亞瑟的忠臣 (Loyal Servant) -- 無特殊能力
- 刺客 (Assassin)         -- 可刺殺梅林
- 莫甘娜 (Morgana)        -- 在派西維爾眼中偽裝成梅林
- 莫德雷德的爪牙 (Minion)  -- 無特殊能力的壞人

完整流程：
[選人數] → [角色配置] → [身份分配(傳閱)] → [夜晚階段(語音)]
→ [隊長組隊] → [全體公投] → [任務執行] → [結果判定]
→ [下一輪/刺殺梅林/遊戲結束]
```

### v1.1: 進階角色 (策略深度)

**預計開發時間**: 2 週
**目標**: 滿足進階玩家的角色多樣性需求

```
新增角色：
- 莫德雷德 (Mordred)      -- 不被梅林看到的邪惡
- 奧伯倫 (Oberon)         -- 邪惡但不知道其他壞人
- 好蘭斯洛特 / 壞蘭斯洛特  -- 互知身份的對立角色

新增功能：
- 角色配置選擇介面（可自由開關進階角色）
- 每個角色的說明卡片
- 推薦配置（根據人數推薦最佳角色組合）
- 討論計時器（可自訂時間）
- 背景氛圍音效（夜晚/白天/投票/任務結果）
```

### v1.2: 湖中女士 + 體驗強化

**預計開發時間**: 2 週
**目標**: 完整官方規則變體支援

```
新增規則：
- 湖中女士 (Lady of the Lake)
  - 第 2/3/4 輪任務結束後觸發
  - 持有者選擇一名玩家查驗陣營
  - 被查驗者不能被再次查驗
  - 語音引導完整查驗流程

新增功能：
- 遊戲歷史記錄（存入 IndexedDB）
- 簡易統計：勝率、常用配置、平均時長
- 多語言語音腳本切換（繁中/英文）
```

### v2.0: 自訂規則引擎

**預計開發時間**: 4 週
**目標**: 讓玩家創建自己的規則變體

```
自訂規則功能：
- 自訂角色（名稱、陣營、夜晚能力描述）
- 自訂任務規則（人數、失敗數量門檻）
- 自訂語音腳本（每個階段的播報文字）
- 自訂規則模板分享（匯出/匯入 JSON）
- 社群規則模板庫（熱門變體規則）
```

### 路線圖時間軸

## 7. 風險與緩解

| 風險 | 嚴重度 | 可能性 | 緩解方案 |
|------|--------|--------|---------|
| Web Speech API 在部分 Android 裝置上 TTS 品質差 | 高 | 中 | 提供語速/音量調整、預先測試語音可用性、提供降級方案（純文字模式） |
| 夜晚階段語音流程複雜，玩家容易搞混 | 高 | 中 | 充分的語音間隔、清晰的視覺提示、每步驟等待確認按鈕 |
| 傳閱手機看身份時被旁邊的人偷看 | 中 | 高 | 設計防窺介面（需點擊才顯示、自動倒數隱藏、方向鎖定提示） |
| 遊戲流程太長導致中途放棄 | 中 | 中 | 儲存遊戲進度、可暫停/繼續、精簡非必要等待時間 |
| 使用者不熟悉 PWA 概念 | 低 | 高 | 提供「加入主畫面」引導、QR Code 直達連結 |

## 8. 與奶酪大盜的協同策略

### 8.1 交叉導流

```
奶酪大盜首頁 → 「更多遊戲」入口 → 阿瓦隆
阿瓦隆遊戲結束 → 「想換個輕鬆的？」 → 推薦奶酪大盜
```

### 8.2 共享基礎設施

| 模組 | 共享內容 |
|------|---------|
| TTS Composable | `useTTS()` 語音引擎 |
| Timer Composable | `useTimer()` 計時器 |
| Audio Composable | `useAudio()` 音效播放 |
| 玩家輸入元件 | 玩家名稱輸入、人數選擇 |
| 遊戲歷史 Store | IndexedDB 持久化 |
| UI 元件庫 | 按鈕、卡片、Modal 等基礎元件 |

### 8.3 品牌統一

兩款遊戲在同一個 App 中，統一的設計語言和品牌體驗，強化「桌遊語音主持人」的平台定位，而非單一遊戲工具。
