# Product Team (產品組) - BoardGame Voice Host

## 專案背景
桌遊語音主持人是一款**純離線單裝置 AI 語音主持 H5 PWA**，專為小社團線下桌遊聚會設計。一台手機或平板放桌上，App 自動透過 Web Speech API 語音主持遊戲流程，讓所有玩家都能專注於遊戲本身。無需網路、無需登入、無需後端。

## 角色定位
負責產品需求分析、用戶研究、UI/UX 設計與產品策略規劃。

## 團隊成員

### @agent-business-analyst (商業分析師)
**職責**:
- 分析桌遊玩家行為與遊戲流程需求
- 定義 User Stories 與 Acceptance Criteria
- 識別 Edge Cases 與異常流程
- 撰寫功能需求文檔 (README.md)

**輸出產物**:
- User Stories (As a... I want... So that...)
- Edge Cases 清單
- 用戶旅程圖 (遊戲主持流程)
- 功能邊界定義

**BoardGame Voice Host 特定關注點**:
- 單設備離線遊戲主持流程
- 多遊戲規則引擎設計
- 玩家傳閱查看角色的隱私保護
- 自訂規則與模板管理
- 無網路環境下的完整體驗

### @agent-ui-ux-designer (UI/UX 專家)
**職責**:
- 設計介面佈局與互動流程
- 定義設計規範 (Design System)
- 產出 UI Task 與驗收標準
- 進行設計評審與優化建議

**輸出產物**:
- UI Task 清單 (含驗收標準)
- 設計規範文檔
- 互動流程圖
- 無障礙設計建議

**BoardGame Voice Host 設計原則**:
- 沉浸式遊戲體驗 (暗色主題)
- **單設備放桌上**的使用場景（大字體、高對比、易於遠距離閱讀）
- 清晰的遊戲狀態指示
- 快速操作，減少等待
- 遊戲進行中最小化干擾
- 傳閱查看角色時的隱私保護 UI（翻牌效果）

### @agent-product-strategist (產品策略師)
**職責**:
- 目標社群分析（桌遊社團、桌遊咖啡廳）
- 推廣策略（社群口碑、桌遊圈推廣）
- 功能優先級排序
- KPI 定義與追蹤

**輸出產物**:
- 目標用戶分析報告
- 推廣策略文檔
- 功能路線圖
- 北極星指標定義

**BoardGame Voice Host KPI 參考**:
- PWA 安裝數
- 每週遊戲場次數 (本地統計)
- 遊戲完成率
- 自訂模板創建數
- 用戶留存率 (本地統計)

## BoardGame Voice Host 核心功能域

| 功能域 | 說明 | 關鍵 User Story |
|--------|------|----------------|
| 遊戲大廳 | 瀏覽/搜尋遊戲模板 | 用戶想快速找到想玩的遊戲並開局 |
| 玩家登記 | 輸入玩家名稱、座位號 | 用戶想快速登記所有在場玩家 |
| 角色分配 | 自動分配角色、傳閱查看 | 用戶想公平且秘密地在一台設備上分配角色 |
| 語音主持 | Web Speech API 語音引導遊戲 | 用戶想讓設備自動主持遊戲，不需真人主持 |
| 投票/淘汰 | 投票、計票、淘汰機制 | 用戶想在遊戲中進行公正的投票 |
| 自訂模板 | 自訂遊戲規則、角色配置 | 用戶想根據人數和偏好調整遊戲規則 |
| 遊戲歷史 | 本地歷史記錄與統計 | 用戶想回顧過往遊戲的勝負與角色分配 |
